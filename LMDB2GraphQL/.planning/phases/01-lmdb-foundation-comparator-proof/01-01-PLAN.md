<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| strfry LMDB env → LMDB2GraphQL reader | strfry is the sole writer; we open its data read-only. A write txn here corrupts the live env. |
| LMDB (C) → heed extern "C" trampoline → Rust Comparator → C++ FFI | Untrusted/possibly-malformed key bytes flow into the C++ comparator on every scan compare. |
| build-time C++ source (vendored golpe) → linked binary | Vendored upstream code is a private-API dependency surface. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | strfry live env (data corruption) | mitigate | `EnvFlags::READ_ONLY` on every open; no `write_txn` helper exists in env.rs (verified by grep gate in Task 1); production never uses `NO_LOCK` |
| T-01-02 | Tampering / DoS | C++ comparator on malformed/too-short key | mitigate | Vendored source replaces `throw` with `std::abort()` (Task 2); compiled `-fno-exceptions` so no exception unwinds across the `extern "C"` boundary (eliminates RESEARCH UB finding); Rust `compare` impls never panic (A6) |
| T-01-03 | Tampering (silent wrong data) | comparator not taking effect → LMDB falls back to memcmp | mitigate | Task 3 control assertion proves the registered comparator changes scan order vs. the no-comparator default; full per-index self-check follows in plan 03 |
