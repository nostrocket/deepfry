## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| strfry LMDB env (fixture/live DB) → reader (range seek) | Bytes on disk are strfry's own, co-located, read-only mounted. The only "input" on this path is strfry's own index keys/values; no external/untrusted client input reaches the seek path. |
| committed golden vectors → seek gate oracle | The static `include_str!` oracle that supplies the expected numerically-correct landing levIds. If misread, the gate could pass vacuously — mitigated by the non-vacuous trip test. |
| process exit semantics | The fail-closed contract depends on `run_comparator_self_check` returning Err propagating through `?` in main to a non-zero exit. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering (silent wrong data) | comparator unregistered / wrong / memcmp fallback at startup — the exact failure mode CR-01 leaves undetected | mitigate | Task 2 adds MDB_SET_RANGE seeks on the Event__kind/Event__pubkeyKind adversarial pairs; the seek consults the registered comparator and must land on the numerically-correct neighbor (levId=2) or run_comparator_self_check returns Err → non-zero exit (LMDB-06/LMDB-05/D-03/D-04) |
| T-04-02 | Repudiation/Spoofing | the new seek gate passes vacuously (e.g. asserts nothing, or reads the wrong field) | mitigate | Task 2 ships a non-vacuous test proving the seek gate TRIPS under a memcmp/absent comparator (memcmp neighbor levId=3 ≠ golpe-correct levId=2); a passing-only test is explicitly disallowed |
| T-04-03 | Tampering | accidental write txn against strfry env via the new seek path | mitigate | Task 1 helper uses only `env.read_txn()` (short, per-call, dropped); no write_txn anywhere; grep acceptance gate confirms `write_txn` count stays 0 and `.create(` count stays 0 (LMDB-01 read-only invariant preserved) |
| T-04-04 | Denial of Service | long-lived RoTxn pinning LMDB pages during the seek | mitigate | The seek helper opens a per-call read txn bounded to a single MDB_SET_RANGE positioning + first-entry read and drops it immediately; no unbounded scan introduced |

No `high`-severity threats are introduced by this change. The change is a read-only startup self-check exercising bounded LMDB range seeks on a co-located, strfry-owned DB; there is no untrusted external input on this path. The one previously-high risk (comparator failing silently → silently-wrong query results downstream) is precisely what T-04-01 now mitigates with a fail-closed seek gate, raising assurance rather than introducing new exposure.
