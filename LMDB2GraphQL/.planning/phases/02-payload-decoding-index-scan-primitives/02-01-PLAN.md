<objective>
Build the EventPayload decoding foundation: add the `zstd` dependency (used by Plan 02),
add the `NostrEvent` + `DecodedEvent` types, open the `EventPayload` and
`CompressionDictionary` sub-DBs read-only with `IntegerComparator`, and implement the
`0x00` (raw JSON) decode path. This is the LMDB-07 deliverable and the contract foundation
that Plan 02 (`0x01` zstd path) extends.

Purpose: Phase 3's query engine hydrates matched levIds into full events via this decoder;
Phase 4 serves the retained raw JSON as an exact passthrough. One decode produces both the
typed struct (field access) and the raw bytes (passthrough) — no double parse (D-01).

Output: `src/lmdb/payload.rs` (open helpers + `decode_event_payload` + `0x00` path +
`PayloadError`), `NostrEvent`/`DecodedEvent` in `types.rs`, `zstd` in `Cargo.toml`,
`tests/payload_test.rs` 0x00 round-trip test.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| strfry LMDB on disk → lmdb2graphql process | Event JSON written by the network (untrusted) crosses into this process when an EventPayload value is read and decoded (D-12) |
| crates.io → build | New `zstd` dependency (and transitive zstd-safe/zstd-sys) enters the build graph |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | `decode_event_payload` 0x00 path | mitigate | `serde_json::from_slice` returns `Err` on malformed JSON → `PayloadError::JsonDecode`; `decode_payload_skip_on_error` skips+warns+counts (D-11). Acceptance: `[0x00]+invalid-json` test returns `Err`, never panics. |
| T-02-02 | Tampering | unknown EventPayload type byte | mitigate | `PayloadError::UnknownTypeTag { tag }` for any byte other than 0x00/0x01 — logged + skipped, no UB. Acceptance: `[0xFF]+...` test returns `UnknownTypeTag`. |
| T-02-03 | Tampering | decoded NostrEvent fields (esp. `content`) treated as data | accept | D-12: decoded JSON is untrusted; no sig re-verification (strfry validated on ingest, D-04). Lenient deserialize ignores unknown fields; downstream phases must not treat known fields as trusted. No code mitigation in this phase beyond structural validation via serde. |
| T-02-04 | Tampering | read-only LMDB invariant | mitigate | All sub-DB opens use `.open()` never `.create()`; `get_event_payload` opens a read txn only (never a write txn). Acceptance: `grep -c '\.create(' src/lmdb/payload.rs` == 0. |
| T-02-SC | Tampering | `cargo add zstd` (npm/pip/cargo-equivalent install) | mitigate | Blocking human checkpoint (Task 1) verifies zstd resolves to crates.io with no override; audit verdict [ASSUMED] requires human sign-off; never auto-approvable. |

Note: the zstd decompression-bomb threat (DoS) and the 0x01 truncated/missing-dict threats are
introduced and mitigated in Plan 02 (the 0x01 path); the `DecompressedTooLarge` /
`TruncatedZstdPayload` / `DictNotFound` PayloadError variants are defined here so the enum is stable.
</threat_model>
