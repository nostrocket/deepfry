<objective>
Implement the `0x01` zstd-dictionary-compressed EventPayload decode path: a lazily-populated,
concurrency-safe `DictCache` keyed by `dictId`, wired into `decode_event_payload` so a `0x01`
payload decompresses with `CompressionDictionary[dictId]` and decodes to the same `DecodedEvent`
shape as `0x00`. This is the LMDB-08 deliverable.

Purpose: strfry stores `0x01` payloads only after an offline `strfry compact`, but the path MUST
be correct when it occurs. The lazy cache (D-09) means default `0x00`-only deployments pay nothing;
the concurrency-safe cache (D-10) supports Phase 4's tokio resolvers decoding `0x01` concurrently.

Output: `DictCache` + the `0x01` branch in `decode_event_payload` (extending Plan 01's `payload.rs`),
a fixed decompression-size ceiling, and a synthetic `0x01` round-trip unit test (no compacted fixture
required — Phase 5 / OPS-03 owns the live-compacted-DB integration test).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| strfry LMDB on disk → process | 0x01 EventPayload bytes (untrusted, network-origin event compressed by strfry) decoded into the process |
| CompressionDictionary sub-DB → zstd decoder | dictId from the payload selects a dictionary; a mismatched/absent dictId must fail closed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Denial of Service | zstd decompression bomb / unbounded output buffer | mitigate | `Decompressor::decompress(frame, MAX_EVENT_DECOMPRESSED_SIZE)` with a fixed 4 MiB ceiling — returns `Err` (not UB, not unbounded alloc) on overflow → `PayloadError::ZstdError`/`DecompressedTooLarge`, then D-11 skip+warn+count. Acceptance: over-ceiling test returns `Err`, no panic. |
| T-02-06 | Tampering | truncated 0x01 payload (< 5 bytes) | mitigate | length guard `raw.len() >= 5` before reading dictId → `PayloadError::TruncatedZstdPayload`. Acceptance: 4-byte payload test returns `TruncatedZstdPayload`. |
| T-02-07 | Tampering | dictId referencing a non-existent dictionary | mitigate | `DictCache::get_or_load` returns `PayloadError::DictNotFound { dict_id }` when the LMDB GET misses; skip+warn+count (D-11). Acceptance: absent-dictId test returns `DictNotFound`. |
| T-02-08 | Tampering | malformed JSON after successful decompression | mitigate | `serde_json::from_slice` on the decompressed bytes returns `Err` → `PayloadError::JsonDecode`; skip+warn+count. |
| T-02-09 | Tampering | read-only LMDB invariant during dict load | mitigate | DictCache slow path opens a read txn only (never a write txn) and drops it before building the dictionary. Acceptance: `grep -c '\.create(' src/lmdb/payload.rs` == 0; write_txn never called. |
| T-02-10 | Tampering | decoded JSON treated as data (no sig re-verify) | accept | D-12 / D-04: decompressed JSON is untrusted; no signature re-verification on the hot path (strfry validated on ingest). Structural validation via serde only. |
