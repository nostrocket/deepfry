<objective>
Generalize Phase 1's proof-of-concept index scans into reusable, bounded, resumable,
direction-aware cursor scan primitives over every `Event__*` index, tested in isolation.
This is the LMDB-09 deliverable.

Purpose: Phase 3's query engine composes these `(composite key, levId)` scans into filter
routing, `latestPerAuthor` (reverse `Event__pubkeyKind` prefix walks), and `(created_at, lev_id)`
cursor pagination. The scan primitive deliberately does NOT hydrate (D-05) — it yields raw
`(key, levId)` pairs so Phase 3 decides which levIds to look up, avoiding over-hydration of
events a later filter discards. Short per-call read txns (D-08) keep strfry able to reclaim
free pages so `data.mdb` does not grow (LMDB-09 / spec §6.4).

Output: `src/lmdb/scan.rs` (`ScanDirection`, bounded forward/reverse scans, the `limit=0`
windowing loop), module wiring, and `tests/scan_test.rs` correctness tests driven by the
committed golden vectors.
</objective>

| Boundary | Description |
|----------|-------------|
| strfry LMDB on disk → process | Event__* index VALUE bytes (8-byte LE levId) read during a scan; a malformed VALUE must not crash the scan |
