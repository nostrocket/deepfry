<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller cursor → scan bounds | the `PageCursor` is decoded (in 03-01) then used to construct scan upper bounds here; bad cursor values must not cause unbounded scans or panics |
| disk payload → NIP-40 decision | expiration is parsed from untrusted decoded event tags; a malformed expiration value must not panic |
| filter limit → scan work | `limit` (and limit==0) drives how much the engine scans; must remain bounded |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-DOS | Denial of Service | `execute_query` over-fetch loop | mitigate | The backfill loop is bounded: it stops at `valid.len() >= limit` OR when merge returns empty; each batch is `DEFAULT_WINDOW_SIZE`-bounded and each per-prefix scan is limit-bounded; limit==0 falls back to DEFAULT_WINDOW_SIZE for v1 (Phase-4 ceiling is the hard cap). No single scan walks the whole DB. |
| T-03-NIP40 | Tampering | `is_expired` tag parse | mitigate | `tag[1].parse::<u64>()` failure is ignored (event treated as non-expiring on unparseable expiration), never unwrapped/panicked; `tag.len() >= 2` guards indexing. |
| T-03-CUR | Tampering | cursor → scan bound | mitigate | Cursor was length/format-validated at decode (T-03-CUR in 03-01); here it only sets a created_at upper bound and an exclusion comparison — out-of-range values simply yield empty/older pages, never out-of-bounds reads. |
| T-03-RDONLY | Tampering | all engine scans | accept | execute_query/latest_per_author only call scan_index_bounded / merge_prefixes / get_event_payload, all MDB_RDONLY short-txn, `.open()` only. Verified by Task 3 grep gates. |
</threat_model>
