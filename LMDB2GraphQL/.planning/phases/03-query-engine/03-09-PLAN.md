<objective>
Close CR-02 and CR-03: route execute_query_internal through a TRUE incremental k-way merge with
a per-stream frontier, and make `since` exhaustion per-stream instead of global.

CR-02: today `execute_query_internal` (engine.rs:267-269) sorts each iteration's combined batch in
ISOLATION, so a stream's iteration-2 entries can be appended AFTER an earlier iteration's
lower-ts entries from another stream — violating (created_at DESC, lev_id DESC) and corrupting the
page cursor. The correct `merge_prefixes` BinaryHeap in merge.rs is ORPHANED (imported nowhere in
the production path — WR-04). Fix: extend merge.rs's heap merge with windowed per-stream refill,
route the engine through it, and DELETE the inline sort. The frontier rule: only emit entries with
(ts, lev_id) >= the low watermark of every non-exhausted stream; buffer the remainder and refill
only the stream(s) whose buffer ran dry.

CR-03: today `since_cutoff` (engine.rs:272-297, break at ~380) is GLOBAL — any stream crossing
`since` terminates ALL streams, silently starving denser streams that still have events >= since.
Fix: per-stream exhaustion — when a stream's guarded, descending batch contains the first entry
with ts < since, truncate the batch at that point and mark THAT stream exhausted; continue the loop
for other streams; termination falls out of all-streams-exhausted.

Also fold in the cleanups in these two files: IN-01 (dead `stream_prefixes` vector + its `drop` in
merge.rs:137-151), IN-02 (empty `if since_cutoff {}` block in engine.rs:293-297), IN-03 (skip_count
accumulated but never surfaced — emit a tracing::debug when > 0 before returning, or remove the
accumulator).

Purpose: the phase goal requires correct ordering and pagination across ALL multi-stream patterns;
these two blockers are the last failed truth in 03-VERIFICATION.md (truth #5, score 3/5).
Output: a windowed k-way merge in merge.rs that the engine uses as its single merge path, with
per-stream since exhaustion, plus regression tests for cross-iteration ordering, page union, and
multi-stream since.
</objective>
