<objective>
Close the sole remaining Phase-3 blocker (new REVIEW CR-01 / VERIFICATION truth #5): `execute_query_internal`
makes exactly ONE `merge_windowed` call with fixed headroom (`emit_limit = 2*limit + 256`) and sets
`next_cursor = None` whenever `valid.len() < limit`. When residual tag/kind/author drops, NIP-40 expiry
runs, or cursor-exclusion on dense seconds exceed the headroom, the query under-returns with no resume
handle and reachable events become silently unreachable.

This plan restores a bounded round-loop with an explicit budget (the property that currently keeps the
WR-03 DoS boundary closed) and a partial-result cursor (so the budget cap never strands reachable events).
It also folds in two cheap hardening items flagged in the VERIFICATION anti-patterns table.

Purpose: pagination correctness — VERIFICATION truth #5 must flip from FAILED to VERIFIED while the
single-call DoS bound (WR-03) is preserved as a round budget.
Output: a looping `execute_query_internal`, a `MAX_ROUNDS` budget constant, a partial-result cursor,
rewritten comments, one new regression test, and two small hardening fixes.

Plans 03-01 through 03-09 are complete and MUST NOT be modified. This plan only edits the three files
listed in `files_modified`.

Rewrite the single-call body of `execute_query_internal` (currently engine.rs:191-309) into a bounded
round-loop while leaving the per-round filter/hydrate/push logic (cursor exclusion, hydration via

Add a module-level constant `const MAX_ROUNDS: usize = 8;` near the top of engine.rs (after the imports,
before is_expired). This is the round budget: total LMDB entries examined per query is bounded by
`MAX_ROUNDS * emit_limit`, which preserves the WR-03 DoS boundary that the single-call model held.

Loop structure (concrete):
- Track a round resume boundary `let mut round_boundary: Option<(u64, LevId)> = cursor_boundary;` —
  initialized from the caller cursor's `(created_at, lev_id)` so round 1 behaves exactly as today.
- Track `let mut rounds: usize = 0;`.
- Each round: build `effective_filter` with `until = Some(round_boundary.map(|(ts,_)| ts).unwrap_or_else(|| filter.until.unwrap_or(u64::MAX)))` — i.e. the resume `until` is the boundary timestamp when set, else the caller's until (or u64::MAX). Re-derive `start_keys = build_start_keys(&effective_filter, &selected, ScanDirection::Reverse)`. If `start_keys.is_empty()` break.
- Call `merge_windowed(env, short_name, &start_keys, batch_size, emit_limit, since)` ONCE per round.
- Compute `let exhausted = merge_batch.len() < emit_limit;` BEFORE consuming merge_batch (true-exhaustion signal per REVIEW CR-01).
- Capture `let last_merged = merge_batch.last().map(|(ts, lev, _)| (*ts, *lev));` BEFORE consuming.
- Run the existing cursor-exclusion + hydrate + residual + push logic against THIS round's merge_batch.
  Cursor exclusion must use `round_boundary` (the advancing exclusion key), so the boundary entry of the
  previous round is never re-emitted. Keep the exclusion comparison identical: drop entries where
  `ts > cur_ts || (ts == cur_ts && lev_id >= cur_lev)`.
- After processing the round: `rounds += 1;`. Break when `valid.len() >= limit || exhausted || rounds >= MAX_ROUNDS`.
- Otherwise set `round_boundary = last_merged;` so the next round rebuilds start_keys with a lowered
  `until` (resume strictly below the last merged entry) and continues.

After the loop, `valid.truncate(limit);` and build `next_cursor`:
- If `valid.len() == limit` → cursor from `valid.last()` (existing behavior — more events likely exist).
- ELSE IF `valid.len() > 0 && !exhausted` (budget cap or a partial round stopped us before the merge was
  truly exhausted) → STILL build a cursor from `valid.last()` so the remainder is reachable. This is the
  core fix: never return `None` when reachable events remain below the page boundary.
- ELSE (`exhausted` and under-full, OR `valid` empty) → `None` (true end of stream).
