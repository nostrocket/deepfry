<objective>
Wire the GraphQL adapter end-to-end: implement the three resolvers (`events`,
`latestPerAuthor`, `stats`) that compose the Phase-3 engine, build the query-only schema
with shared `AppState`, mount it on an axum router, and extend `main.rs` to start the HTTP
server after the existing startup gates pass. Consumes the type contracts from Plan 04-01.

Purpose: Deliver the read-only GraphQL endpoint satisfying API-01..API-06 — pure adapter
work calling already-built engine functions; no query semantics are reimplemented.

Output: A running `POST /graphql` endpoint (+ GraphiQL on `GET /graphql`) serving events,
latestPerAuthor, and stats over strfry's live LMDB.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → GraphQL API (`POST /graphql`) | Untrusted query input (filter fields, cursor, limit, hex ids/authors) crosses into resolvers |
| resolver → engine (LMDB) | Resolver-built `NostrFilter`/`PageCursor` cross into the read-only engine |
| engine error → client response | `QueryError` crosses back toward the HTTP response (must not leak internals) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-DOS | Denial of Service | `events` / `latestPerAuthor` resolvers | mitigate | Hard limit ceiling 500, default 100 clamped at the resolver BEFORE calling the engine (D-04/D-05/Pitfall 6); engine's own MAX_ROUNDS=8 bound caps scan work; cursor pagination on (created_at, lev_id) prevents whole-DB scans (API-05). **D-06: NO async-graphql `.limit_depth()`/`.limit_complexity()` for v1** — the limit ceiling + MAX_ROUNDS bound is the deliberate DoS guard and the schema is shallow; executors MUST NOT add depth/complexity limiters (deferred to v2 if exposed to untrusted public traffic) |
| T-04-FANOUT | Denial of Service | `latestPerAuthor` author-count fan-out | accept | `authors.len()` deliberately uncapped per D-08 (user's explicit choice); each author bounded at `perAuthor`≤500; total work = authors×perAuthor; flagged for Phase-5/v2 observability (CONTEXT Deferred) |
| T-04-CUR | Tampering | `after` cursor decode | mitigate | `PageCursor::decode` is fail-closed (T-03-CUR, Phase 3); malformed cursor → `CursorDecode` → client-facing `INVALID_CURSOR` error, never a panic |
| T-04-LEAK | Information Disclosure | `map_query_error` (Lmdb/Payload variants) | mitigate | `Lmdb`/`Payload` errors logged via `tracing::error!` and returned as opaque `"internal error"`; only `CursorDecode` is client-facing — LMDB/payload internals never reach the response |
| T-04-HEX | Information Disclosure | hex decode of `ids`/`authors` | mitigate | Malformed-hex inputs surface a generic client error without echoing the offending bytes |
| T-04-WRITE | Elevation of Privilege | GraphQL schema surface | mitigate | `Schema::build(Query, EmptyMutation, EmptySubscription)` — no Mutation root registered; structurally impossible to write (D-10/API-06); resolvers use `read_txn` only, never `.create()`/`write_txn` (T-03-RDONLY) |
| T-04-SC | Tampering | npm/pip/cargo installs | mitigate | No new package installs in this plan (deps added + audited in Plan 04-01); no [ASSUMED]/[SUS] packages to gate |
</threat_model>
