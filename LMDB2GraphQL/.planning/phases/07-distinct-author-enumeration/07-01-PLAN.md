<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| engine → strfry LMDB | LMDB2GraphQL reads strfry's live, externally-owned `Event__pubkey` index. strfry is the sole writer; a second writer corrupts the env. |
| caller `after` cursor → engine | The `after: Option<&[u8;32]>` arrives already-decoded from the resolver (Plan 07-02 owns the hex→bytes fail-closed decode). At this layer it is a fixed-width 32-byte value — no parsing. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-RDONLY | Tampering | distinct_authors LMDB access | mitigate | Open `Event__pubkey` via `open_index_string_uint64` (.open, never .create); `env.read_txn()` only, never `write_txn`; enforced by the read-only grep gate in both task acceptance criteria. |
| T-07-TXN | Denial of Service | RoTxn lifetime | mitigate | Single short per-call RoTxn opened inside the fn and dropped at end of scope (LMDB-09 / D-08) — strfry can reclaim free pages; no txn held across pages (each page is a separate call). |
| T-07-DOS | Denial of Service | enumeration cost | mitigate | Each call does at most `limit` seeks (one B-tree seek per distinct pubkey); the resolver clamps `limit` ≤ 500 (Plan 07-02). Seek-skip is O(distinct authors), not O(total events). |
| T-07-SHORTKEY | Information Disclosure | malformed index key | accept | A sub-32-byte `Event__pubkey` key is a structural surprise from the externally-owned DB; logged via `tracing::warn!` and the loop breaks (no panic, no slice OOB). Low risk: strfry never writes such keys. |
</threat_model>
