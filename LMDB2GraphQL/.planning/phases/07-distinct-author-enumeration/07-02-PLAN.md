## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GraphQL client → resolver | The `after` and `limit` args are untrusted client input crossing into the service. `after` is hex-decoded fail-closed; `limit` is clamped. |
| resolver → engine | The resolver passes a validated `[u8;32]` and a clamped `limit` to the read-only engine fn; no LMDB access happens at the resolver layer beyond spawn_blocking. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-CUR | Information Disclosure | `after` cursor decode | mitigate | Decode hex fail-closed; reject non-hex / wrong-length with a generic client error that does NOT echo the offending bytes (mirror T-04-CUR/T-04-HEX). Test 4 asserts no-echo. |
| T-07-DOS | Denial of Service | `limit` arg | mitigate | Clamp `limit` to [1,500] default 100 at the resolver before calling the engine — identical to events() (D-04/D-05). Each page bounded to `limit` seeks. |
| T-07-LEAK | Information Disclosure | engine QueryError surfacing | mitigate | Reuse `map_query_error`: Lmdb/Payload errors are logged and returned as opaque "internal error"; only INVALID_CURSOR is client-facing. |
| T-07-SEND | Tampering | RoTxn !Send across spawn_blocking | mitigate | Clone `env` BEFORE the closure (Pitfall 1); the engine opens/drops its own short RoTxn inside spawn_blocking. |
| T-07-MUT | Elevation of Privilege | mutation surface | mitigate | No mutation is added; schema remains EmptyMutation. Test 5 asserts the SDL has no `type Mutation` (API-06 preserved). |
| T-07-SC | Tampering | dependency installs | accept | No new crates added — AuthorsPage/resolver/tests reuse existing deps (async-graphql, heed, tokio). No package-legitimacy gate needed. |
