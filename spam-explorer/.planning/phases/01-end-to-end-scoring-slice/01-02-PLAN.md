## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-02 | Tampering | internal/dgraph/resolve.go (--seed interpolation) | mitigate | Interpolate the seed with `%q` / `strconv.Quote` (web-of-trust idiom), never raw `%s` — quotes/escapes the string into the DQL literal. Full hex-format validation is Phase 3 (CLI-02). |
| T-01-03 | Information disclosure | internal/dgraph queries | mitigate | Queries select ONLY uid, pubkey, follows — never event content; data-separation rule enforced by query design (ID-only graph) |
| T-01-04 | Tampering (write-back) | internal/dgraph client | mitigate | Read-only client only (D-06): no Mutate/Commit/Alter code path exists; client uses NewReadOnlyTxn exclusively |
| T-01-05 | Denial of service | gRPC receive on ExpandFrontier | accept | Local-only, operator-run, one-shot CLI; the 256 MiB cap (T-01-SC headroom) bounds a single frontier response. Full-graph memory bounding is Phase 2 (INGEST-02). |
| T-01-SC | Tampering | go mod / dependency installs | mitigate | dgo/v210 + grpc pinned (Plan 01) to versions already building in the web-of-trust sibling; RESEARCH Package Legitimacy Audit verdict OK for both — no checkpoint required |

ASVS L1; no high-severity threats — read-only, local-only, ID-only graph. The only mutation/injection surfaces are mitigated above; DoS is accepted for a local one-shot tool.
