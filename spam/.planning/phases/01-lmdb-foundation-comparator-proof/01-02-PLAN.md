## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Upstream Docker registry → pinned image | An unpinned `:latest` lets the parent stack drift to a different strfry/dbVersion silently. |
| Operator config (~/deepfry/lmdb2graphql.yaml) → process | map_size / db path values control which env is opened and whether it opens at all. |
| Vendored golpe.yaml + seed → fixture/golden vectors | The correctness oracle depends on these inputs being the pinned, hand-audited truth. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering / Elevation | strfry image drift (unpinned :latest changes dbVersion) | mitigate | Pin parent Dockerfile.strfry + config to a full sha256 digest (Task 3, D-09); plan 03's dbVersion==3 gate fails closed if a drifted image is mounted |
| T-02-02 | Spoofing (false correctness) | golden vectors dumped from the same comparator we ship | mitigate | D-05: golden vectors hand-computed analytically from comparator semantics, never from a golpe-linked tool — independent ground truth (Task 5) |
| T-02-03 | Tampering | config file overwrite during tests | mitigate | Never write ~/deepfry/; tests use tempfile::tempdir() (Task 3, CLAUDE.md) |
| T-02-04 | Tampering | non-deterministic fixture → CI assertion meaningless | accept-with-fallback | A5 spike: if `strfry import` is not byte-deterministic, PROVENANCE.md mandates semantic comparison (events present + scan order) instead of byte-exact sha256 (A5 measured in Task 4, recorded in Task 5 PROVENANCE); risk is low (no live attacker, CI-internal) |
| T-02-SC | Tampering | dockurr/strfry image provenance | mitigate | Full sha256 digest pin (immutable content address); strfry git commit cross-referenced and recorded in PROVENANCE.md |

No threat here is rated `high` for blocking once the digest pin (T-02-01) is applied. The crate-install legitimacy gate (T-01-SC) is satisfied by Task 1's blocking-human checkpoint.
