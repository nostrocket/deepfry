## Decisions Made

- **Adversarial timestamp choice:** 1700000255/1700000256 (Nov 2023 timestamps) provide the `0xFF→0x00` byte inversion at a valid timestamp range; avoids far-future timestamps that could be rejected by relay.
- **kind=2 instead of kind=3:** Nostr kind=3 (contact list) is a replaceable event — strfry would delete the earlier event on import. Used kind=2 (non-replaceable, non-standard) to keep all 11 events in the fixture.
- **Fixed test key:** `sk=0x01` (secp256k1 generator point G) — universally known as a test key; never used with real funds.
- **A5 policy:** Since import is BYTE-IDENTICAL, CI uses exact sha256 comparison; PROVENANCE.md documents both the sha256 and the fallback (semantic comparison) for future reference if strfry changes.
