<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| strfry LMDB env → reader (Meta + indexes) | Untrusted on-disk bytes (Meta fields, index keys) are parsed and ordered; a wrong version/endianness or unregistered comparator yields silently-wrong results. |
| committed golden vectors → self-check | The oracle the gate trusts; if the self-check reads them wrong, it could pass vacuously. |
| process exit semantics | The fail-closed contract depends on errors actually terminating the process. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering / Elevation | wrong dbVersion or endianness (drifted/foreign DB) | mitigate | assert_db_version (==3) + assert_endianness fail closed before any index access; main returns Err → non-zero exit (Task 1/3, LMDB-02/03, V7) |
| T-03-02 | Tampering (silent wrong data) | comparator mis-registered → memcmp fallback / wrong comparator on wrong index | mitigate | Self-check scans all six indexes and asserts full-sequence equality vs. golden vectors; adversarial fixture guarantees memcmp produces a different order → mismatch caught (Task 3, D-04/D-06) |
| T-03-03 | Tampering | oversized/truncated/malformed Meta read | mitigate | Bounds-checked slice parse (try_into on fixed offsets) returns Err rather than panicking; A3-verified offsets; read-only env prevents write corruption (Task 1, V5) |
| T-03-04 | Repudiation/Spoofing | self-check passes vacuously (reads no entries) | mitigate | Integration test asserts the check FAILS on a mutated golden vector, proving it is not a vacuous pass (Task 3) |
| T-03-05 | Tampering | accidental write txn against strfry env | mitigate | Only env.rs read-only helpers used; no write_txn anywhere; production path uses READ_ONLY (no NO_LOCK) — clippy/grep confirm (Task 3, LMDB-01) |

The version/endianness gate failing OPEN would be a `high` severity issue; T-03-01 mitigates it directly with a fail-closed Err→exit path verified by tests. No residual `high` threats.
</threat_model>
