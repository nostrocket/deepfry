<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filter values → key bytes | hex id/pubkey strings and tag values from a caller are decoded into LMDB scan key bytes; malformed values must not panic or produce out-of-bounds reads |
| index → merge | `(key, levId)` pairs read from LMDB are trusted-shape (validated by Phase-2 scan's <8-byte guard) but the merge must still bound work by `limit` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-DOS | Denial of Service | `merge_prefixes` per-prefix scans | mitigate | Each per-prefix `scan_index_bounded` is bounded by `limit` (or DEFAULT_WINDOW_SIZE when limit==0); the heap emits at most `limit` results. No unbounded scan. Verified by Test 3 (length ≤ limit). |
| T-03-HEX | Tampering | `build_start_keys` hex decode | mitigate | Unparseable id/pubkey hex is skipped with `tracing::warn!`, not unwrapped; `created_at_from_key` uses `saturating_sub` + `try_into().unwrap_or` so short keys never panic. |
| T-03-RDONLY | Tampering | all scans | accept | This plan only calls `scan_index_bounded`, which takes `&heed::Env` (never `&RoTxn`) and opens MDB_RDONLY short txns; no `.create()`, no write txn introduced. |
</threat_model>
