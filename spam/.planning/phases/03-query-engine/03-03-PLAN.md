<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| EventPayload bytes (disk) → decoded event | payload bytes read from strfry's LMDB may be malformed/corrupt; decode must fail closed per-event without aborting the query |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-PAY | Tampering | `hydrate_lev_ids` decode loop | mitigate | A corrupt/undecodable payload is caught, logged with `tracing::warn!(lev_id, reason)`, counted in `skip_count`, and skipped — the batch continues (D-11). decode_event_payload* never panics on malformed input (Phase-2 guarantee). |
| T-03-DOS | Denial of Service | hydration batch size | accept | hydrate_lev_ids hydrates exactly the levIds it is handed; the merge/engine layer bounds that count by `limit` (mitigated upstream in 03-02 / 03-04). |
| T-03-RDONLY | Tampering | get_event_payload | accept | Hydration delegates to `get_event_payload`, which opens MDB_RDONLY short txns and never `.create()`s; no write path introduced. |
</threat_model>
