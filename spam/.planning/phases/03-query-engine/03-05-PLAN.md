<objective>
Close gap Cluster 2 (CR-05): the positional `zip` between `filtered_batch` (ts, lev_id keys) and
`hydrated` (a possibly-shorter `Vec<DecodedEvent>`) in `execute_query_internal` misaligns every
event after the first corrupt-payload skip, corrupting the PageCursor and result ordering. The
designed-for D-11 degraded path (skip corrupt payload, keep serving) silently breaks pagination.

Fix the root cause by making `hydrate_lev_ids` carry the `lev_id` association explicitly so a
length mismatch can never desync the caller, then join on `lev_id` in both `execute_query` and
`latest_per_author`.

Purpose: Restore the QRY-04 / D-11 contract — a single corrupt payload degrades gracefully
without corrupting the cursor or the (created_at, lev_id) keys of surviving events.
Output: New `hydrate_lev_ids` signature, lev_id-join in engine.rs, a test-only writable-env helper
that injects a corrupt payload, and a regression test that fails under the old positional-zip code.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LMDB EventPayload bytes → decoder | A physically-present but corrupt/unknown-tag payload value crosses into the decoder |
| query result → (future) GraphQL caller | A cursor derived from misaligned keys would let a caller skip or re-read events |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-CR05-I | Information Disclosure | execute_query cursor | mitigate | Join hydrated events on lev_id; build cursor only from authoritative filtered_batch (ts, lev_id) so a skipped payload cannot shift the emitted keys or leak/skip events on the next page |
| T-03-CR05-D | Denial of Service | hydrate skip path | accept | A corrupt payload is skipped (skip_count++), never panics, never aborts the batch (D-11); bounded by batch window |
| T-03-SC | Tampering | cargo installs | mitigate | No new dependencies added by this plan; if any appear, slopcheck + blocking human checkpoint required |
</threat_model>
