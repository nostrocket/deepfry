<objective>
Close gap Cluster 1 (CR-01, CR-02, CR-03, CR-04 + WR-01, IN-01, IN-04): `execute_query` returns
silently wrong events. Reverse scans walk past the filter prefix into lexicographically smaller
prefixes (CR-01); `filter.since` is never enforced (CR-02); the DUPSORT batch-boundary windowing
re-implements the broken `Included`-restart pattern Phase-2 specifically replaced (CR-03); and the
stuck-window advance keeps a stale lev_id (CR-04). Duplicate filter values double every result
(WR-01).

Fix by: bounding every per-prefix stream to its own prefix in `merge_prefixes`; enforcing `since`
as a stop bound and adding a post-hydration kind/author/id residual in `execute_query_internal`;
replacing the `Included`-restart windowing with the proven key-granular exclusive-resume pattern
already in `scan.rs` (`collect_window` / `scan_index_windowed`); resetting lev_id to u64::MAX on
the stuck-window advance; and deduping start keys.

Purpose: Restore QRY-01 — `events()` selects the correct index and returns correctly ordered,
correctly bounded, non-duplicated results.
Output: prefix-guarded merge, since-bounded + residual-filtered engine, exclusive-resume windowing,
deduped start keys, and regression tests that exercise non-lowest-kind, since bounds, dup groups
of size >=3, and duplicate authors.
</objective>

## Trust Boundaries

|----------|-------------|
| filter values → index scan | Untrusted GraphQL callers (Phase 4) supply kinds/authors/ids/since/until; wrong routing returns events the filter excludes |
| LMDB reverse scan → result set | A reverse scan with no prefix bound walks the entire index below the start key — incorrect-data-exposure surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-CR01-I | Information Disclosure | merge_prefixes reverse scan | mitigate | take_while starts_with prefix guard + post-hydration kind/author/id residual ensure no wrong-prefix event is ever returned |
| T-03-CR02-I | Information Disclosure | since enforcement | mitigate | since read as a stop-bound + per-candidate drop; results never include events older than since |
| T-03-CR03-D | Denial of Service / data loss | DUPSORT windowing | mitigate | Key-granular exclusive-resume (proven in scan.rs) so no dup is dropped or re-emitted; loop bounded by limit/window |
| T-03-WR01-I | Information Disclosure | duplicate start keys | mitigate | Dedup start keys so duplicate filter values cannot double or halve the page |
| T-03-DOS | Denial of Service | over-fetch loop | accept | Each batch is window-bounded; loop stops at valid.len()>=limit or stream exhaustion (unchanged invariant) |
| T-03-SC | Tampering | cargo installs | mitigate | No new dependencies; slopcheck + blocking human checkpoint if any appear |
