# Quick Task 260615-1e6: Fixture-free comparator self-check Summary

Redesigned the startup comparator self-check to validate correctness against any live strfry
DB using the physical B-tree order as the oracle — no fixture, no golden vectors, no unbounded
log dumps — so it no longer fails closed (and emits ~243MB to stderr) on real databases.

## What changed

### `src/lmdb/self_check.rs` (rewritten)
- **Removed** the embedded golden-vector `include_str!` block, the `GoldenVectorJson` /
  `GoldenVectors` struct + `load_committed` / `get` / `mutate_reverse`, the
  `build_adversarial_seeks` fixture-keyed seeks, and the `OrderMismatch` / `GoldenVectorLoad`
  / `GoldenVectorMissing` error variants.
- **Phase A — FFI/logic correctness.** Per `Event__*` index, walk physical order via
  `db.iter()` collecting KEYS over a bounded sample (`SELF_CHECK_SAMPLE_CAP = 10_000`). For
  each adjacent pair assert `Cmp::compare(prev, cur) != Ordering::Greater` (non-decreasing;
  DUPSORT → adjacent equal keys valid). Violation ⇒ `MonotonicityViolation` (fail closed).
- **Phase B — registration / memcmp-fallback detection.** Collect *witness pairs* (adjacent
  physical pairs where `memcmp(prev,cur) == Greater` while physical order is `prev < cur`),
  prioritize their keys, and for each do `db.range(K..).next()`, asserting the first key
  `== K`. Mismatch ⇒ `SeekRoundTripMismatch`. If an index yields zero witnesses, a
  `tracing::warn!` records that registration could not be adversarially verified on that
  dataset (round-trip still held). When no witnesses exist, a bounded spread of plain-key
  round-trips is still exercised.
- `Event__created_at` runs both phases with `heed::IntegerComparator`.
- New error variants format only: index name, truncated-hex key pair (`first8…last8 (len=N)`
  via `hex_trunc`, capped at `2 * HEX_EDGE = 16` bytes), and integer positions. No code path
  can emit an unbounded vector.
- Signature changed to `pub fn run_comparator_self_check(env: &heed::Env) -> Result<(), SelfCheckError>`.
- Per-index short-lived `read_txn()`, bounded by the sample cap; read-only throughout
  (`MDB_RDONLY`); no write txn opened.

### `src/main.rs` (step 8 caller)
- Dropped `GoldenVectors::load_committed()`; now calls `run_comparator_self_check(&env)` with
  fail-closed `?` propagation. Updated the surrounding comment to describe Phase A/Phase B.

### `src/lib.rs`
- No change needed — `lib.rs` re-exports `pub mod lmdb` only; it never re-exported
  `GoldenVectors`.

### Tests
- `tests/self_check_test.rs`: positive test (`Ok(())` on the fixture) and a non-vacuous
  Phase B control proving that a memcmp-fallback open makes the exact-seek round-trip on the
  kind=256 witness key miss (the `SeekRoundTripMismatch` condition).
- `src/lmdb/self_check.rs` unit tests: `hex_trunc` bounding, a reversed comparator tripping
  `MonotonicityViolation` (Phase A non-vacuity, old T-03-04 spirit), and the fixture
  `Event__kind` yielding ≥1 witness pair (Phase B genuinely exercised).
- `tests/comparator_hook_smoke.rs`: unchanged (does not reference the removed symbols).
