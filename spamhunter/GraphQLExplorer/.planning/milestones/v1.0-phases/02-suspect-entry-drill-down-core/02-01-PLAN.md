<objective>
Install `nostr-tools` (exact-pinned, behind a legitimacy human-verify gate) and build the pure
`identifier/` module: the single entry point that turns any pasted string into a discriminated
`ParseResult` — a normalized lowercase-hex pubkey + both display forms, OR an explicit parse
failure. This is the foundation of the ID-03 honesty distinction: a parse failure is the ONLY
thing this module produces as an error; "valid identifier, zero matching events" is decided later
by the query (plan 02-02), never here. A typo must never read as a clean author.

Purpose: Lock the identifier contract (npub/nprofile/hex accept; note/nsec/garbage reject) as
pure, unit-tested logic with zero transport coupling, so the entry bar (02-02) and any future
batch import (Phase 4) reuse one verified normalizer.
Output: `src/identifier/identifier.ts` + `src/identifier/identifier.test.ts`, `nostr-tools` pinned
in package.json/lock.
</objective>
