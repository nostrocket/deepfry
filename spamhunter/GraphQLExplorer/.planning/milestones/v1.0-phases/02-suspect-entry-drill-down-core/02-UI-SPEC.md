# Phase 2 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This is the **second UI surface** of the project. Phase 1 locked the foundational design system (tokens, type scale, color roles, copy voice). **This phase INHERITS that contract verbatim and EXTENDS it** — it does not relitigate spacing, typography, color, or font choices. The effort here is on the NEW surfaces' state treatments, copy, and the two load-bearing honesty contracts (window-size denominator + forgeable-`createdAt` caveat).

---

## Surface Scope (Phase 2)

Phase 2 delivers the app's core value as a vertical slice and introduces the app's **first routing** (a second view). It is still a read-only forensic/utilitarian tool for a single local analyst. The honesty posture is now load-bearing: a partial window must never read as exoneration, and a `createdAt`-derived signal must never be presented as ground truth.

New surfaces this contract covers:

1. **Suspect entry bar** — a paste bar mounted in the **app shell/header** (the stats dashboard remains home/landing). Accepts `npub` / `note` / `nprofile` bech32 OR a 64-char hex pubkey. Must visibly distinguish a **parse failure** (inline validation error) from a **valid-but-zero-match author** (neutral empty state) — ID-03.
2. **Author drill-down view** — a new route (author lowercase hex in the URL, e.g. `#/a/<hex>`). Header shows **both identifier forms**: the human-friendly bech32 (`npub…`) AND the 64-char hex, both as mono, both copyable as escaped plaintext.
3. **Event timeline** — newest-first across **all** kinds (no kind filter). Each row: `kind`, `createdAt` (rendered as both a human-readable UTC time and the raw epoch), and a short single-line escaped-plaintext content preview. A **"Load more"** control widens the window via cursor pagination.
4. **Posting-rate / burst panel** — hand-rolled CSS/SVG rate bars (NO chart lib). **Asymmetric** interpretation: burst present = suspicious; burst absent ≠ clean. A **PERSISTENT, non-dismissible** "createdAt is author-claimed and forgeable" caveat sits beside the rate chart.
5. **Window-size honesty indicator** — a **NON-REMOVABLE** indicator on **every signal surface**: "computed over N fetched events · hasMore · time range". A partial window is never framed as a clean bill of health (DRILL-05).

There are **no destructive user actions** in this phase (the API and tool are read-only). Entering a new suspect / loading more / navigating are all non-destructive reads.

## Typography

**Inherited verbatim from Phase 1.** Two families (mono for data/IDs, sans for chrome), exactly two weights (400 / 600). No new roles introduced — Phase-2 surfaces map onto the existing roles as follows.

| Role | Size | Weight | Line Height | Family | Phase-2 usage |
|------|------|--------|-------------|--------|---------------|
| Display | 28px | 600 | 1.2 | mono | reserved for hero data (e.g. the drill-down's fetched-event count `N` in the window indicator, if shown large) — use sparingly; the drill-down hero is the identity, not a single number |
| Heading | 20px | 600 | 1.2 | sans | drill-down view title ("Author"), state-shell headings (parse error, zero-match) |
| Body | 16px | 400 | 1.5 | sans | state-shell body copy, the forgeable-`createdAt` caveat text, empty-state body |
| Label | 13px | 600 | 1.5 | sans | timeline column captions (Kind / Time / Content), panel titles, window-indicator label text, badge/caption text (uppercase optional, letter-spacing ≤ 0.04em) |

**Phase-2 data rendering rules (extend the mono-for-data discipline):**
- The **hex pubkey** and **bech32 `npub`** in the identity header render in **mono** (`--font-mono`) as data, not prose. Long hex should wrap with `word-break: break-all` (or be truncated with a copy-full affordance) so it never overflows.
- A timeline row's **`kind`** and **`createdAt`** (both the human time and the raw epoch) render in **mono** — they are data. The **content preview** renders in **sans body** (it is human prose) at 13–16px, single-line, truncated with ellipsis, **escaped plaintext**.
- Large integers (event count `N` in the window indicator) use `Intl.NumberFormat` locale grouping, consistent with Phase 1.

## Color

**Inherited verbatim from Phase 1.** Same 60/30/10 discipline, same tokens. Phase 2 does **not** add new base or semantic colors. It only (a) extends the accent reservation list **minimally and explicitly**, and (b) maps the existing semantic colors onto the new states.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0E1116` (`--bg`) | App background, drill-down page shell |
| Secondary (30%) | `#1A1F29` (`--surface`) | App-shell/header bar (now hosts the paste bar), identity header card, rate/burst panel, timeline-row hover surface |
| Accent (10%) | `#3DD6C0` (`--accent`) | see reservation list below — extended minimally |
| Text primary | `#E6E9EF` (`--text`) | identity values, panel headings, timeline data |
| Text muted | `#9AA4B2` (`--text-muted`) | labels, captions, the window-indicator text, content-preview-of-empty |
| Border / divider | `#2A313D` (`--border`) | card/panel borders, timeline row dividers, paste-bar input border, rate-bar baseline/axis |

**Accent reserved for (explicit — extends the Phase-1 list; NEVER "all interactive elements"):**

1. *(Phase 1, unchanged)* The "corpus changed" nudge (badge/banner) — only on the dashboard.
2. *(Phase 1, unchanged)* The live-poll active indicator dot.
3. **(Phase 2, NEW)** The **suspect-entry submit affordance** — the single primary action that takes the analyst into a drill-down. This is the app's one "go" action; it earns the accent. Implemented as either an accent-bordered submit button or an accent focus treatment on the paste bar — pick one, not both. The paste-bar input itself stays neutral (`--surface` fill, `--border` edge); only the submit affordance carries accent.

**Explicitly NOT accent in Phase 2** (stay neutral chrome — `--surface` fill, `--border` edge, `--text`/`--text-muted` text, matching Phase 1's `.refreshButton`):
- The **"Load more"** button (it is a secondary/repeatable action, not the primary entry — neutral).
- Timeline rows, row hover, the back/home navigation control, the copy-pubkey affordance.
- The rate/burst bars themselves (see rate-panel color rule below).

**Semantic state colors** (inherited, meaning-bound — NOT decorative). Phase-2 mapping:

| State | Token | Meaning in Phase 2 | Reads as |
|-------|-------|--------------------|----------|
| Connecting / info | `--connecting` #5AA9E6 | drill-down first load gated on `/ready` / `503` backoff (reuses Phase 1 connecting shell) | "working / warming up" — never an error |
| Recoverable / retryable | `--recoverable` #E0A24E | `INVALID_CURSOR` (pagination restarts from page 1), `413` (page too large — shrink), `503` after first connect, validation messages | "self-correcting / clear next step" |
| Hard failure | `--hard-fail` #E5534B | code-less internal error, network/relay unreachable (reuses Phase 1 copy with `VITE_GRAPHQL_URL`) | "genuine failure — generic, no internals leaked" |

**Rate/burst-panel color rule (load-bearing honesty constraint):**
The rate bars are **neutral by default** (`--text-muted` fill on a `--border` baseline). Because burst interpretation is **asymmetric**, color must NOT imply a verdict:
- A detected burst window MAY be tinted with **`--recoverable` (amber)** to draw the eye to a *suspicious* pattern — amber, not red, because a burst is a signal-to-investigate, never a proven verdict. It is always paired with a text label ("burst") and the shape of the spike itself (color is not the sole signal).
- The **absence** of any amber/spike MUST remain visually neutral and MUST NOT read as "green/clean." There is **no green, no success color** anywhere in this panel. Quiet bars = "no burst detected in this window," nothing more.
- The teal accent is **never** used in the rate panel.

Destructive: **none** — no destructive user actions (read-only tool). No destructive-confirmation color is introduced.

Color is never the sole carrier of meaning: parse-error vs zero-match, burst vs no-burst, and every transport state each pair color with a text label and/or shape (dot, badge, bar spike, inline message).

## State Treatments (Phase 2 interaction contract)

Each state is **distinct and non-blank**. The two ID-03 states (parse failure vs zero-match) and the window-honesty indicator are the highest-stakes treatments — an analyst's judgment depends on never confusing them.

### Suspect entry (the paste bar)

| State | Visual treatment | Color role |
|-------|------------------|------------|
| **Idle / empty** | Neutral paste-bar input in the shell header, placeholder copy (see Copywriting). Submit affordance carries accent. No error, no result. | Neutral + accent (submit only) |
| **Valid pending** | On a parseable identifier, the input edge stays neutral; submit is enabled. (Optional: a small muted "looks like npub/hex" affirmation — neutral, never green.) | Neutral |
| **Parse failure (ID-03 — "couldn't parse")** | **Inline validation error directly under/beside the input**, amber/recoverable tone with a shape (e.g. a `!` glyph or amber edge on the input) + text: "Not a valid npub/note/nprofile or 64-char hex." The input **stays on the dashboard** — it does NOT navigate, does NOT show an empty timeline. This must be unmistakably an *input* error, never a *result*. | Recoverable (amber) |

### Drill-down view

| State | Visual treatment | Color role |
|-------|------------------|------------|
| **Connecting (first load)** | Reuse the shared `ConnectingShell` (Phase 1) — centered, info-blue, "Connecting to relay…". Distinct from any error. | Connecting (blue) |
| **Loaded — identity header** | A `--surface` card showing both forms: the bech32 `npub…` (mono) and the 64-char hex (mono), each with a copy affordance and a clear "npub" / "hex" label. Title "Author" (sans heading). A back/home control returns to the dashboard. | Neutral |
| **Loaded — timeline (≥1 event)** | Newest-first list. Each row: `kind` (mono), `createdAt` as human UTC + raw epoch (mono), single-line content preview (sans, escaped, truncated). Rows divided by `--border`; subtle `--surface` hover. The window indicator sits **above or below** the list and is always present. | Neutral |
| **Valid identifier, ZERO matching events (ID-03 — "valid but empty")** | A **neutral, calm empty state** inside the drill-down (NOT a parse error, NOT amber/red): heading "No events for this author" + body explaining it is a valid pubkey with zero stored events in this corpus — **and the window indicator still shows** ("computed over 0 fetched events · no more pages"). A typo can never reach this state (it fails at parse); this state means the identifier was genuinely valid. | Neutral / muted text |
| **Load more — idle** | Neutral "Load more" button below the timeline, shown only while `hasMore` is true. | Neutral |
| **Load more — loading** | Button shows a busy/disabled affordance ("Loading…"), inert; no spinner library — text + disabled state. Existing rows stay on screen. | Neutral |
| **Load more — end of window** | When `hasMore` is false, the button is replaced by a muted caption: "End of available events — this is the full window." (Stated as a fact, still framed by the window indicator; NOT "all clean.") | Neutral / muted |
| **`INVALID_CURSOR`** | Inline amber note: "Pagination expired — reloading from the top." Pagination restarts from page 1 (constant filter, verbatim cursor). Informational, auto-recovers. | Recoverable (amber) |
| **`413` page too large** | Amber note: "Request too large — trimming and retrying." (Page size already 100; if hit, shrink and retry.) | Recoverable (amber) |
| **`503` / not ready (after first connect)** | Amber backoff note: "Relay is warming up — retrying…" | Recoverable (amber) |
| **Validation error** | Amber note showing the server's validation message verbatim (user-safe per contract §7). | Recoverable (amber) |
| **Internal error (code-less)** | Red hard-failure: "Something went wrong reading this author's events. Retrying shortly." Generic — never echo server internals. | Hard failure (red) |
| **Network / relay unreachable** | Red hard-failure naming the actual endpoint: "Can't reach the relay at {VITE_GRAPHQL_URL}. Is it running and is VITE_GRAPHQL_URL pointing at it?" | Hard failure (red) |

### Window-size honesty indicator (DRILL-05 — NON-REMOVABLE, on every signal surface)

| Property | Contract |
|----------|----------|
| **Presence** | **Always rendered** wherever a signal/conclusion is shown (the timeline header AND the rate/burst panel). It has **no dismiss control** — there is no state in which it is hidden. It renders even when `N = 0`. |
| **Content** | "Computed over **N** fetched events · {hasMore ? "more available — partial window" : "full window"} · {time range: oldest → newest `createdAt` in the window}". `N` uses locale grouping. Time range shown as human UTC. |
| **Tone / color** | Neutral muted (`--text-muted`) when `hasMore` is false (full window). When `hasMore` is true, the "partial window" segment is emphasized with **recoverable amber** text/shape so the reader cannot miss that the window is incomplete — a partial window must never read as exoneration. |
| **Placement** | Co-located with the signal it qualifies — the analyst reads the conclusion and its denominator together, never separately. |
| **Copy posture** | Always frames count as a denominator ("over N fetched"), never a bare verdict. Pairs with the asymmetric-burst caveat (below) on the rate panel. |

### Forgeable-`createdAt` caveat (DRILL-01 — PERSISTENT, beside the rate chart)

| Property | Contract |
|----------|----------|
| **Presence** | **Always rendered beside the rate/burst panel.** Non-dismissible. There is no state in which the rate chart appears without this caveat. |
| **Content** | A muted body-copy line: "Timing is author-claimed and forgeable — a burst suggests suspicious automation, but quiet timing does not prove the author is clean." |
| **Asymmetry encoding** | The panel's visual + copy must make burst = suspicious-signal and no-burst = inconclusive (NOT clean). No success/green state. The caveat is the textual half of the color-is-not-sole-signal rule for this panel. |
| **Tone / color** | Neutral muted text. The caveat itself is not an error — it is a permanent epistemic disclaimer. |
