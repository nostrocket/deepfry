<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → lens (`/graphql`) | Direct cross-origin request over wildcard CORS to an unauthenticated read-only API; only backend-controlled scalars cross back |
| build tooling → lens (codegen) | Node fetch of introspection at build time (no browser, no CORS); output is a trusted build artifact |
| operator → base URL config | `VITE_GRAPHQL_URL` decides which host the client trusts; default is loopback |

## STRIDE Threat Register (ASVS L1; block on `high`)

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering (XSS) | App.tsx rendering `pinnedStrfryVersion` (backend String) | mitigate | Render as escaped plaintext via JSX interpolation; never `dangerouslySetInnerHTML`. React escapes by default — do not opt out. (V5) |
| T-01-02 | Tampering (defense-in-depth) | `transport/config.ts` base URL | mitigate | Default `VITE_GRAPHQL_URL` to loopback `127.0.0.1:8080`; document in `.env.example`/README that overriding to a public/untrusted host sends queries to and trusts responses from that origin. (V14) |
| T-01-03 | Information Disclosure | Exposing the unauthenticated, wildcard-CORS, full-introspection API beyond loopback | accept | v1 is local-dev only (contract § 10); network placement is the only access control. No production path in v1. Documented in SKELETON.md / .env.example. |
| T-01-SC | Tampering | npm installs (react/urql/graphql/codegen/vite) | accept | RESEARCH § "Package Legitimacy Audit" dispositioned every package Approved; all SUS flags are `too-new` publish-date artifacts on canonical first-party packages with no slopsquat signals and null postinstall — no `[ASSUMED]`/`[SUS]` requiring a human checkpoint. |

No `high`-severity threats in this phase (no user-supplied input rendered yet; loopback default; read-only API).
</threat_model>
