# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This is the **first UI surface** of the project. The tokens, type scale, color roles, and copy voice declared here are **foundational** — the whole app (Phases 2–4) inherits them. Keep it lean, but treat every token as a baseline the later phases extend rather than relitigate.

---

## Surface Scope (Phase 1)

This phase is a **read-only corpus-stats dashboard** for a single local spam-investigation analyst. It is a forensic/utilitarian tool, NOT a consumer app. The UI must read as information-dense, trustworthy, and honest — it never over-states certainty (the project's "window-honesty" posture applies even on the dashboard).

Surfaces this contract covers:

1. **Stats dashboard** — four live scalar values: `eventCount`, `maxLevId`, `dbVersion`, `pinnedStrfryVersion`.
2. **Cold-start "connecting to relay…" state** — gated on `GET /ready`; `503` = retry-with-backoff. Reads as *connecting*, never as a generic error.
3. **Distinct, non-blank error states** per transport `extensions.code` (`INVALID_CURSOR`, `TOO_MANY_AUTHORS`, internal, validation) + HTTP-status errors (`503`, `413`, network) + a generic transport-error fallback. (Only `stats` flows in Phase 1, but the state set is built complete so Phases 2–4 inherit it.)
4. **Non-intrusive "corpus changed" nudge** — driven by polling `maxLevId` diffs. Passive, dismissible. Must NOT aggressively auto-refetch.

There are **no destructive user actions** in this phase (the API and tool are read-only).

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — hand-rolled plain CSS (CSS Modules or a single tokens stylesheet) |
| Icon library | none required for Phase 1 (state dots/badges are CSS; if an icon is needed, inline SVG only — no icon-font/lib dependency) |
| Font | System sans stack for labels/chrome; system monospace stack for numeric/ID values |
| Theme | Dark default, single theme, no toggle in Phase 1 |

**Rationale (locked by user + PROJECT.md):** shadcn was explicitly declined. A four-scalar analyst utility does not justify Radix + Tailwind tooling, and a component library would bias the surface toward a consumer aesthetic. The project constraint is "lightweight CSS/SVG bars, no heavy chart lib." Registry safety gate: **not applicable** (no shadcn, no third-party registries).

**Font stacks (no web-font download — system fonts only):**
- Sans: `system-ui, -apple-system, "Segoe UI", Roboto, sans-serif`
- Mono: `ui-monospace, "SF Mono", "JetBrains Mono", "Fira Code", Menlo, Consolas, monospace`

## Typography

Two families: **mono for data** (numeric stats and IDs read as data, never prose), **sans for chrome** (labels, headings, copy). Weights are restricted to exactly two: 400 (regular) and 600 (semibold).

| Role | Size | Weight | Line Height | Family | Used for |
|------|------|--------|-------------|--------|----------|
| Display | 28px | 600 | 1.2 | mono | The four stat values (`eventCount`, `maxLevId`, `dbVersion`, `pinnedStrfryVersion`) — the data is the hero |
| Heading | 20px | 600 | 1.2 | sans | Dashboard title, state-shell headings ("Connecting to relay…", error titles) |
| Body | 16px | 400 | 1.5 | sans | State-shell body copy, error explanations, nudge text |
| Label | 13px | 600 | 1.5 | sans | Stat labels ("Event count", "Max levId"), badge text, status captions (uppercase optional, letter-spacing ≤ 0.04em) |

Notes:
- `pinnedStrfryVersion` is a `String!` — render in the Display mono slot as **escaped plaintext** (React default escaping; never `dangerouslySetInnerHTML`).
- Large integers (`eventCount`, `maxLevId`) should use locale grouping (`Intl.NumberFormat`) so they stay legible in the mono Display slot.

## Color

Dark-first, single theme. 60/30/10 discipline: one dominant surface, one raised-card surface, one accent reserved to two specific elements. Semantic colors are layered on top and are reserved strictly to state meaning.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0E1116` | App background, page shell |
| Secondary (30%) | `#1A1F29` | Stat cards, raised panels, the dashboard header bar |
| Accent (10%) | `#3DD6C0` (teal) | **Only**: the "corpus changed" nudge, and the live-poll "active" indicator dot |
| Text primary | `#E6E9EF` | Stat values, headings |
| Text muted | `#9AA4B2` | Labels, captions, secondary copy |
| Border / divider | `#2A313D` | Card borders, dividers, hairlines |

**Accent reserved for (explicit — never "all interactive elements"):**
1. The "corpus changed" nudge (badge/banner).
2. The live-poll active indicator dot (shows polling is running / tab visible).

Nothing else uses the accent. Stat cards, headings, and chrome stay neutral.

**Semantic state colors** (meaning-bound, NOT decorative):

| State | Value | Meaning | Reads as |
|-------|-------|---------|----------|
| Connecting / info | `#5AA9E6` (muted blue/cyan) | Cold-start `/ready` gate; `503` backoff retry in progress | "Working / warming up" — NEVER red, NEVER an error |
| Recoverable / retryable | `#E0A24E` (amber) | `INVALID_CURSOR`, `503` backoff after first connect, transient retryable transport issues | "Self-correcting, no action needed (or a clear next step)" |
| Hard failure | `#E5534B` (red) | Code-less `internal error`, unrecoverable transport/network failure | "Genuine failure — generic, no internals leaked" |

Destructive: **none** — there are no destructive user actions in this phase (read-only tool). The red above is for *system* hard-failure state, not a destructive-action confirmation.

Color is never the sole carrier of meaning: every state pairs its color with a text label and/or shape (dot, badge, icon glyph) so the distinction survives for color-blind users and in screenshots.

## State Treatments (Phase 1 interaction contract)

Each state is **distinct and non-blank** — no blank panel, no `TypeError`-on-null. These map 1:1 to the transport classifier's discriminated union.

| State | Visual treatment | Color role |
|-------|------------------|------------|
| **Connecting (cold start)** | Centered shell, Heading "Connecting to relay…", Body sub-copy, animated info-blue indicator. No stat cards shown yet. Distinct from any error. | Connecting/info (blue/cyan) |
| **Loaded / live** | Four stat cards (2×2 grid ≥ md width, stacked single-column below), each: Label (sans 13/600 muted) + Value (mono 28/600 primary). Header shows live-poll accent dot when polling and tab visible. | Neutral + accent dot |
| **Empty corpus (eventCount = 0)** | Render the cards normally with value `0`; show a calm caption "No events in corpus yet." This is **NOT an error state** — neutral styling, no amber/red. | Neutral / muted text |
| **Corpus changed (nudge)** | Non-intrusive accent badge/banner near the header: "Corpus changed — refresh to update." Dismissible. Does **NOT** auto-refetch; clicking it (or a refresh control) is the only thing that re-pulls. | Accent (teal) |
| **Poll paused (tab hidden)** | Accent dot goes neutral/dimmed (no animation); optional muted caption "Paused (tab hidden)". No error. | Neutral / muted |
| **`INVALID_CURSOR`** | Inline amber note: "Pagination expired — reloading from the top." (Auto-recovers; informational.) | Recoverable (amber) |
| **`TOO_MANY_AUTHORS`** | Amber state: "Too many authors in one request — narrowing the batch." (Scaffolded; first exercised Phase 4.) | Recoverable (amber) |
| **`503` / not ready (after first connect)** | Amber backoff note: "Relay is warming up — retrying…" (bounded backoff). | Recoverable (amber) |
| **`413` payload too large** | Amber note: "Request too large — trimming and retrying." (Scaffolded; Phases 2/4.) | Recoverable (amber) |
| **Validation error (code-less message)** | Amber note showing the server's validation message verbatim (it is safe, user-facing). | Recoverable (amber) |
| **Internal error (code-less "internal error")** | Red hard-failure state: "Something went wrong reading the corpus. Retrying shortly." Generic — **never echo raw server internals.** | Hard failure (red) |
| **Network / relay unreachable** | Red hard-failure state with a setup hint: "Can't reach the relay. Is the relay up on 127.0.0.1:8080 (and VITE_GRAPHQL_URL pointing at it)?" | Hard failure (red) |

## Copywriting Contract

Voice: **honest, terse, forensic.** Plain present-tense, no marketing adjectives, no false reassurance, no over-claiming. State what is happening and (if the user can act) the next step. Empty/zero is described as a fact, not a failure.

| Element | Copy |
|---------|------|
| Dashboard title | Corpus stats |
| Primary CTA | Refresh stats (the only action; re-pulls `stats` on demand — there is no other CTA this phase) |
| Stat label — eventCount | Event count |
| Stat label — maxLevId | Max levId |
| Stat label — dbVersion | DB version |
| Stat label — pinnedStrfryVersion | Pinned strfry version |
| Connecting heading | Connecting to relay… |
| Connecting body | Waiting for the relay to report ready. This can take a moment on cold start. |
| Empty state heading | No events in corpus yet |
| Empty state body | The relay is reachable and reporting zero stored events. Nothing to investigate yet. |
| Corpus-changed nudge | Corpus changed — refresh to update. |
| Poll-paused caption | Paused (tab hidden) |
| Error — INVALID_CURSOR | Pagination expired — reloading from the top. |
| Error — TOO_MANY_AUTHORS | Too many authors in one request — narrowing the batch. |
| Error — 503 not ready | Relay is warming up — retrying… |
| Error — 413 payload too large | Request too large — trimming and retrying. |
| Error — validation | (server message, shown verbatim) |
| Error — internal | Something went wrong reading the corpus. Retrying shortly. |
| Error — network/relay unreachable | Can't reach the relay. Is the relay up on 127.0.0.1:8080 (and VITE_GRAPHQL_URL pointing at it)? |
| Destructive confirmation | not applicable — no destructive actions in this phase (read-only tool) |
