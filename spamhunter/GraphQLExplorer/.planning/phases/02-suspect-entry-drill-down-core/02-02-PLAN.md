<objective>
Deliver the thin end-to-end vertical slice of the core value: paste a suspect → land on their
drill-down → read a newest-first timeline against an honest, non-removable window-size denominator
→ widen the window with Load more. This is the app's FIRST routing (a second view) and the first
exercise of Phase 1's opaque-cursor discipline against a real query. The rate/burst panel is the
NEXT slice (02-03) — this plan deliberately ships the timeline + indicator + pagination first so a
real user can do something new after it: inspect any author's event window honestly.

Purpose: Wire the verified `identifier/` module (02-01) and Phase 1's transport boundary (client,
classify, opaque cursor) into two new surfaces (shell entry bar + author drill-down) plus a tiny
hash router, with the DRILL-05 window indicator baked in from the first signal (never retrofitted).
Output: EventsDocument, useHashRoute, useAuthorWindow, WindowIndicator, SuspectEntryBar,
AuthorDrillDown (+ CSS modules), and a modified App.tsx that mounts the shell + router.
</objective>
