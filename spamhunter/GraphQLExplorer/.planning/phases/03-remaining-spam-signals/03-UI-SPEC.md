# Phase 3 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This is the **third UI surface** of the project. Phases 1 and 2 locked the foundational design system (tokens, type scale, color roles, copy voice) and the load-bearing honesty contracts (non-removable window denominator, asymmetric "absence ≠ clean" framing, escaped-plaintext rendering). **This phase INHERITS that contract verbatim and EXTENDS it** — it does NOT relitigate spacing, typography, color, font choices, or the design-system tool decision. The effort here is on the THREE new stacked signal panels and the lazy raw-JSON inspector: their state treatments, copy, and how each one carries the inherited honesty posture.

---

## Surface Scope (Phase 3)

Phase 3 broadens the existing **`AuthorDrillDown`** view into the full forensic picture by **stacking three new client-side signal panels** below the existing identity header / timeline / posting-rate panel, plus a **lazy, on-demand raw-JSON inspector** for any single event. There is **no new route, no new entry mechanic, no new pagination mechanic** — all of that shipped in Phase 2 and is consumed read-only. All three panels re-derive over the **same accumulated `useAuthorWindow` window** and re-compute live on "Load more" (Phase 2 pattern).

New surfaces this contract covers:

1. **Near-duplicate content panel (DRILL-02)** — surfaces clusters of exact-duplicate and near-duplicate posts (two-stage: exact normalized-hash bucket → shingle/Jaccard ≈0.8). Results are **always framed against the window denominator** ("3 of 50 fetched are near-duplicates"), never a bare "0 duplicates". Includes a per-timeline-row cluster affordance plus a cluster summary in the panel.
2. **Tag / mention fan-out panel (DRILL-03)** — aggregates `p` (mentions / fan-out), `e` (event references), `t` (hashtags / stuffing) over the window: top-N most-mentioned pubkeys, top-N hashtags, each with counts, plus a **per-event outlier flag** for an event carrying an unusually high tag count.
3. **Kind-distribution panel (DRILL-04)** — a hand-rolled CSS/SVG bar histogram of the author's event kinds (NIP/kind name where known + raw kind number), with a count of **out-of-safe-range `kind` / `createdAt`** values flagged (never silently mis-computed or dropped).
4. **Lazy raw-JSON inspector (DRILL-04)** — opens the canonical `raw` bytes for ONE selected event, fetched **lazily on demand** (`events(filter: { ids: [id] }, limit: 1) { events { raw } }` — the only place `raw` is ever fetched), rendered as **escaped plaintext** in a `<pre>` (pretty-printed if it parses as JSON, verbatim otherwise), **never** executed as HTML/markdown.

Each of the three panels carries the **non-removable `WindowIndicator`** (DRILL-05 carried forward). There are **no destructive user actions** in this phase (read-only API and tool): viewing a cluster, opening raw bytes, and loading more are all non-destructive reads.

**Out of scope (do not design here):** batch / multi-author triage (Phase 4), cross-author duplicate detection (single-author only this phase), any server-side or persisted analysis, any new author-entry or pagination UI.

## Typography

**Inherited verbatim from Phases 1–2.** Two families (mono for data/IDs/raw, sans for chrome/prose), exactly two weights (400 / 600). **No new roles introduced** — Phase-3 surfaces map onto the existing roles.

| Role | Size | Weight | Line Height | Family | Phase-3 usage |
|------|------|--------|-------------|--------|---------------|
| Heading | 20px | 600 | 1.2 | sans | (unchanged) drill-down view title; not re-used per-panel |
| Body | 16px | 400 | 1.5 | sans | panel caveat / disclaimer lines (e.g. the "absence ≠ clean" framing on each panel), raw-inspector explanatory copy |
| Label | 13px | 600 | 1.5 | sans | **panel titles** ("Duplicate content", "Tags & mentions", "Event kinds"), column captions, count-chip labels, cluster-badge text, window-indicator label, kind-bar axis labels, outlier-flag label |
| Data | 13px | 400 | 1.5 | mono | counts/denominators in cluster summaries, kind numbers, mentioned-pubkey hex (truncated mono), tag counts, the window-indicator numbers, **and the raw-JSON `<pre>` body** |

**Phase-3 data rendering rules (extend the mono-for-data discipline):**

- **Raw-JSON inspector body** renders in **mono** (`--font-mono`) inside a `<pre>` at 13px / line-height 1.5, with `white-space: pre-wrap` + `word-break: break-all` so long lines never overflow the panel. It is **data, escaped**, never prose, never executed.
- **Mentioned-pubkey** entries in the tag panel render the pubkey in **mono**, truncated (e.g. first/last 8 chars) with the full value available on hover/title, mirroring the identity-header hex treatment. Counts render mono.
- **Hashtags** (`t`-tag values) are author-supplied strings: render as **escaped plaintext** in mono (they are tag data, not prose), single-line, truncated with ellipsis. Counts render mono.
- **Kind labels**: the human NIP/kind name (where known) renders in **sans label** (13px/600); the raw kind number renders in **mono** beside it. Unknown kinds show the number alone, in mono, with a muted "(unknown kind)" sans label.
- **Cluster counts and "X of N fetched" denominators** use `Intl.NumberFormat` locale grouping (consistent with Phases 1–2) and render in mono.

## Color

**Inherited verbatim from Phases 1–2.** Same 60/30/10 discipline, same tokens. Phase 3 adds **no new base or semantic colors** and **does NOT extend the accent reservation** — `--accent` (teal) remains reserved for the "Inspect author" submit + dashboard nudge/poll dot ONLY and appears NOWHERE in any signal panel or the raw inspector.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0E1116` (`--bg`) | App background, drill-down page shell |
| Secondary (30%) | `#1A1F29` (`--surface`) | All three signal-panel cards, raw-inspector panel/`<pre>` surface, top-N list-row hover, cluster-detail surface |
| Accent (10%) | `#3DD6C0` (`--accent`) | **NOT used in any Phase-3 surface** — reservation unchanged (see below) |
| Text primary | `#E6E9EF` (`--text`) | panel data values, kind names, mentioned-pubkey hex, raw-JSON body |
| Text muted | `#9AA4B2` (`--text-muted`) | panel titles, labels, captions, window-indicator text, neutral histogram bars, "absence ≠ clean" framing, empty-cluster body |
| Border / divider | `#2A313D` (`--border`) | panel borders, histogram baseline/axis, top-N row dividers, raw-`<pre>` border, cluster-detail dividers |

**Accent reserved for (UNCHANGED — Phase 3 extends it by ZERO):**

1. *(Phase 1)* The "corpus changed" nudge — dashboard only.
2. *(Phase 1)* The live-poll active indicator dot — dashboard only.
3. *(Phase 2)* The **"Inspect author" submit affordance** — the app's one "go" action.

**Explicitly NOT accent in Phase 3** (stay neutral chrome — `--surface` fill, `--border` edge, `--text` / `--text-muted` text, matching the existing `.loadMore` / `.copyButton`):

- The **cluster expander / row cluster badge**, the **raw-inspector trigger and close controls** — all neutral chrome.
- The **kind-histogram bars**, the **top-N mention/hashtag lists** — neutral by default (see panel color rules below).

**Semantic state colors** (inherited, meaning-bound — NOT decorative). Phase-3 mapping:

| State | Token | Meaning in Phase 3 | Reads as |
|-------|-------|--------------------|----------|
| Recoverable / signal-to-investigate | `--recoverable` #E0A24E | a detected suspicious pattern worth the analyst's eye: a near-duplicate cluster, a mass-mention/hashtag-stuffing outlier flag, a per-event high-tag-count outlier, and the "partial window" segment of the window indicator | "look here — suspicious, not yet a verdict" |
| Recoverable / retryable | `--recoverable` #E0A24E | the lazy raw-fetch failed and is retryable (network/`503`), `INVALID_CURSOR` on Load more (inherited) | "self-correcting / clear next step" |
| Hard failure | `--hard-fail` #E5534B | the lazy raw-fetch hit a code-less internal error or the relay is unreachable | "genuine failure — generic, no internals leaked" |

**Panel color rules (load-bearing honesty constraints — apply to all three new panels):**

- **Asymmetric, never exonerating.** A panel's bars/lists/clusters are **neutral by default** (`--text-muted` fill on a `--border` baseline). A *detected* suspicious pattern MAY be tinted **`--recoverable` (amber)** to draw the eye — always paired with a text label + a shape (dot / badge / bar emphasis). The **absence** of any amber MUST remain visually neutral and MUST NOT read as "green/clean." There is **no green, no success color, no teal** anywhere in any panel.
- **Near-duplicate panel:** a cluster of ≥2 near-dupes is amber-tinted (badge + count + "near-duplicate" label). Zero clusters renders as a neutral muted fact ("No near-duplicates among the 50 fetched events"), never green, never "clean".
- **Tag/mention panel:** top-N lists are neutral; only a flagged **outlier** (an event with an unusually high tag count, or a mention/hashtag count over the `thresholds.ts` mass/stuffing default) is amber-tinted, paired with an "outlier" / "high fan-out" / "stuffing" label + dot.
- **Kind histogram:** bars are neutral `--text-muted`. The **out-of-safe-range count**, when > 0, is shown as an amber flagged note ("N events with out-of-range kind/timestamp") with a dot — never silently dropped. A zero-out-of-range corpus shows nothing amber. Kinds themselves are never "good" or "bad" colored — they are neutral data.

Destructive: **none** — no destructive user actions (read-only tool). No destructive-confirmation color is introduced.

Color is never the sole carrier of meaning: every cluster, outlier flag, out-of-range count, partial-window segment, and raw-fetch error pairs its color with a text label and/or a shape (dot, badge, bar emphasis, inline message).

## State Treatments (Phase 3 interaction contract)

Each state is **distinct and non-blank**. Every panel renders its `WindowIndicator` even when its result is zero/empty — a zero result framed against the denominator is a *fact*, never a clean bill of health.

### Near-duplicate content panel (DRILL-02)

| State | Visual treatment | Color role |
|-------|------------------|------------|
| **Has clusters** | `--surface` panel, 13px muted title "Duplicate content". A summary line ("**3 of 50 fetched** are near-duplicates across **2** clusters" — counts mono, framed against the denominator). Each cluster shown as a neutral group with an **amber cluster badge** (dot + "near-duplicate" / "exact duplicate" label + member count), expandable to list its member events (escaped single-line content previews, mono kind + time). Timeline rows belonging to a cluster also carry a small neutral cluster affordance linking to the cluster. Window indicator co-located. | Neutral + recoverable (amber on clusters) |
| **Zero clusters** | Neutral muted fact: "**No near-duplicates** among the {N} fetched events." Window indicator still present. NOT amber, NOT green, NOT "clean". Asymmetry note rendered (see below). | Neutral / muted |
| **N = 0 (empty window)** | "Computed over 0 fetched events — no content to compare." Window indicator present (N = 0 state). | Neutral / muted |
| **Asymmetry framing (persistent)** | A muted body line on the panel: "Duplicate content is one spam signal among several — its absence does not clear this author." Always rendered, never dismissible. | Neutral / muted |

### Tag / mention fan-out panel (DRILL-03)

| State | Visual treatment | Color role |
|-------|------------------|------------|
| **Has tags** | `--surface` panel, title "Tags & mentions". Two top-N lists: **Most-mentioned pubkeys** (`p`) and **Top hashtags** (`t`), each row a mono value (truncated pubkey / escaped hashtag) + a mono count, neutral dividers. `e`-references summarized as a count. A per-event **outlier flag** (amber badge + "high fan-out" / "stuffing" label + dot) appears where an event's tag count exceeds the `thresholds.ts` default. Window indicator co-located. | Neutral + recoverable (amber on outliers) |
| **No tags in window** | Neutral muted fact: "**No p/e/t tags** among the {N} fetched events." Window indicator present. NOT amber, NOT green. | Neutral / muted |
| **N = 0 (empty window)** | "Computed over 0 fetched events — no tags to aggregate." | Neutral / muted |
| **Asymmetry framing (persistent)** | Muted body line: "High mention/hashtag fan-out is suspicious; low fan-out does not clear this author." Always rendered. | Neutral / muted |

### Kind-distribution panel (DRILL-04)

| State | Visual treatment | Color role |
|-------|------------------|------------|
| **Has events** | `--surface` panel, title "Event kinds". **Hand-rolled CSS/SVG bar histogram** (mirrors `RatePanel`: neutral `--text-muted` bars on a `--border` baseline). Each bar labeled with the NIP/kind name (sans label) where known + the raw kind number (mono); unknown kinds show the number + a muted "(unknown kind)" label. Per-bar count shown (mono). Window indicator co-located. | Neutral |
| **Out-of-safe-range present** | An **amber flagged note** above/beside the histogram: "**N events** with out-of-range kind/timestamp" (dot + label). These events are **counted and flagged**, never silently dropped or mis-bucketed (reuses Phase 2 `isSaneTs` / bounds-check discipline). | Recoverable (amber) |
| **N = 0 (empty window)** | "Computed over 0 fetched events — no kinds to show." Window indicator present. | Neutral / muted |

### Lazy raw-JSON inspector (DRILL-04)

| State | Visual treatment | Color role |
|-------|------------------|------------|
| **Trigger (idle)** | Each timeline row carries a neutral chrome control "View raw" (matches `.copyButton` styling — `--surface` / `--border`, 13px semibold). `raw` is **never** fetched until this is activated. | Neutral |
| **Fetching** | The trigger shows a busy/disabled affordance "Loading raw…" (text + disabled, no spinner library). The rest of the row/panel stays interactive. | Neutral |
| **Loaded — JSON** | The canonical bytes shown in a `--surface` `<pre>` (mono, 13px, `white-space: pre-wrap; word-break: break-all`, `--border` 1px, `--space-md` padding), **pretty-printed** (2-space) because the bytes parse as JSON. A neutral "Close" control collapses it. A muted caption notes "Canonical bytes — escaped, not executed." Rendered via React text node escaping; **never** `dangerouslySetInnerHTML`. | Neutral |
| **Loaded — non-JSON / unparseable** | Same `<pre>`, shown **verbatim** (not pretty-printed) as escaped plaintext, with a muted caption "Raw bytes shown verbatim (not valid JSON)." | Neutral |
| **Fetch error — retryable** | Amber inline note in place of the `<pre>`: "Couldn't load raw bytes — retrying." (network / `503` / `INVALID` paths reuse the Phase 2 `classify()` boundary.) | Recoverable (amber) |
| **Fetch error — hard** | Red inline note: "Couldn't load the raw bytes for this event." Generic — never echo server internals. | Hard failure (red) |
| **Zero-match (id returns no event)** | Neutral muted note: "No canonical bytes returned for this event id." (A valid-but-absent fact, not an error.) | Neutral / muted |

### Window-size honesty indicator (DRILL-05 — NON-REMOVABLE, on EVERY new panel)

| Property | Contract |
|----------|----------|
| **Presence** | **Always rendered on each of the three new panels** (duplicate, tags, kinds). It has **no dismiss control**. It renders even when the panel's result is zero and when `N = 0`. The raw inspector is a per-event drill, not a window-wide signal, so it does not carry the indicator — but it inherits the escaped-plaintext + asymmetry posture. |
| **Content** | Reuses the Phase 2 `WindowIndicator` component verbatim: "Computed over **N** fetched events · {partial/full window} · {oldest → newest}". `N` uses locale grouping. |
| **Tone / color** | Neutral muted when `hasMore` is false; the "partial window" segment emphasized with recoverable amber + dot when `hasMore` is true — a partial window must never read as exoneration. |
| **Placement** | Co-located with each panel's result, so the analyst reads each conclusion and its denominator together. |
| **Copy posture** | Every panel frames its result as "X of N fetched", never a bare verdict. |
