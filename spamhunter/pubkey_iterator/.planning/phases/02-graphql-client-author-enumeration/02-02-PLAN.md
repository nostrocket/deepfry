## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client Ôćĺ LMDB2GraphQL adapter (HTTP) | untrusted JSON response crosses back into the process |
| operator Ôćĺ client (endpoint URL) | operator-supplied endpoint string configures where requests go |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Information disclosure | Error logging of response bodies | mitigate | Do not log full/large response bodies (V7); the `ClientError` carries only HTTP status + the first error `message` string, not the whole body. |
| T-02-05 | Tampering / SSRF | Operator-supplied endpoint URL | accept | Endpoint is operator-supplied (default loopback `http://127.0.0.1:8080/graphql` per contract ┬ž10), not user input ÔÇö no SSRF surface in Phase 2. Forward constraint: a future config-driven URL (OPS-03 / Phase 4) MUST validate scheme/host. Documented, not a Phase-2 blocker. |
| T-02-06 | Tampering | In-body GraphQL `errors[]` ignored Ôćĺ acting on bad/empty `data` | mitigate | `query<T>` checks `env.errors` BEFORE `env.data` (contract ┬ž7 / D-08); in-body errors are surfaced as `ClientError::Graphql`, never silently dropped. |
| T-02-07 | Denial of Service | Unbounded response Ôćĺ memory | accept | Page `limit` ÔëĄ500 (contract clamp) bounds each `authors` response (~35 KB); the client holds one page at a time. |
