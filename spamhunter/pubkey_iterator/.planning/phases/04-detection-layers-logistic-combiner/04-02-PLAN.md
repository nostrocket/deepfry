## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| event content → L1/L3 | `Event.content` is UNTRUSTED; both layers treat it as opaque text (tokenize/hash/count only; never execute or fetch). |
| fingerprint rows → SQLite | persist via the single-writer actor with `params![]`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | fingerprint UPSERT | mitigate | `UPSERT_FINGERPRINT` binds every value with `params![]` / `?N`; content_hash/simhash are integers, never interpolated. |
| T-04-02 | Repudiation | non-deterministic SimHash | mitigate | hand-rolled FNV-1a (deterministic by construction); NO ahash/SipHash/randomized hasher (Pitfall 1); determinism test asserts equal u64 across calls; persisted `fingerprint.simhash` is reproducible. |
| T-04-05 | DoS | adversarial / huge content crashing a layer | mitigate | events capped (≤100, 256 KiB body); SimHash/entropy are O(content len) and bounded; shingle/cluster loops are over ≤100 events (O(n²) ≈ 5000 comparisons max). |
| T-04-06 | Integrity | signed-ordering the u64-as-i64 hashes | mitigate | content_hash/simhash compared for equality/Hamming only; never signed-numeric-ordered (model.rs caveat). |
</threat_model>
