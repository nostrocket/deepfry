<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| this engine → whitelist :8081 | read-only HTTP consumer of `GET /check/{pubkey}`; loopback, no creds; never decides membership, only consumes `{whitelisted:bool}`. |
| event content/tags → L4 | UNTRUSTED; L4 parses URLs for host strings only — never fetches them. |
| whitelist_url (config) | operator-supplied; an attacker-controlled URL would be an SSRF concern but the URL is operator-trusted config, not end-user input (forward concern — note for a future service mode). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-07 | Tampering (of verdicts) | whitelist unreachable → mass false-positive | mitigate | on transport error emit L0 sub-score 0.0 (fail toward not-flagging) + evidence `unreachable`; NEVER map an outage to "absent" (Pitfall 2). |
| T-04-08 | Information Disclosure | logging whitelist response / pubkeys | mitigate | do not log full response bodies; evidence stores only the boolean, not raw HTTP. |
| T-04-09 | SSRF | operator-supplied whitelist/adapter URL | accept (forward concern) | URLs are operator-trusted config, not user input; L4 parses URLs but never fetches them. Documented for a future service/multi-tenant mode. |
| T-04-04 | Elevation/Integrity | enforcement side-effect | mitigate | SCORE-04 — no whitelist write, no quarantine, no event mutation; the no-enforcement test asserts only score/signal/fingerprint/weight tables change. |
| T-04-02 | Repudiation | non-determinism via cache TTL | mitigate | per-run no-TTL cache (A7); fixed layer order; OPS-02 covered by Plan 01/02 re-run tests. |
