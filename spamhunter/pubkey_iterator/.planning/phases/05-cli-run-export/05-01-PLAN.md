<objective>
Lay the foundation slice for the Phase-5 CLI: add the one Phase-5 dependency
(`clap`), add the `suspected_spammer` materialization table to the schema, and
resolve the run-lifecycle unification (RESEARCH Open Q1 / A5) so a single run_id
spans enumeration and scoring with the τ + weight snapshot attached to it.

This is the seam every later slice plugs into: Plan 02's `run` orchestration
needs `enumerate::run` to NOT own the run lifecycle, and Plan 03's `export` needs
the `suspected_spammer` table + a write connection that does not violate the
single-writer invariant. None of this is a horizontal layer — it is the
load-bearing wiring change (per D-04/D-06 reproducibility contract) that the
`run` end-to-end test in Plan 02 will pin.

Purpose: Make the canonical scoring run own the snapshot + the `done` mark, so
`export` finds suspected rows on the same run the operator names (Pitfall 1/2).
Output: `clap` in Cargo.toml; `suspected_spammer` table; widened `enumerate::run`
signature; `Store::set_run_config_json` + `Store::export_write_conn`.
</objective>
