<objective>
The `export` slice — turn a completed run's scores into the shippable, reviewable
suspected-spammer list, materialized into SQLite (D-05: SQLite only, no flat
file). `export` is a pure-SQL materialize of an already-computed flag: the
`score.suspected` column already records "score > τ at run time", so `export`
runs one idempotent `INSERT … SELECT … FROM score WHERE run_id=? AND suspected=1`
into the `suspected_spammer` table, stamping τ (from the run's config_json
snapshot) and a review rank. Per-layer reasons are NOT duplicated — they stay
JOINable from `signal` (D-05), so a reviewer reads the verdict with any SQLite
client and joins to the evidence.

After this plan the phase deliverable is complete: `cargo run -- run` produces
scores, `cargo run -- export` materializes the reviewable list, and a reviewer can
`SELECT … FROM suspected_spammer JOIN signal USING (run_id, pubkey)`.

Purpose: Deliver SCORE-03 — the suspected-spammer list with per-layer evidence,
exportable from SQLite, point-in-time-correct per run (D-06).
Output: `src/export.rs` (`materialize_suspected`, `latest_done_run`), the wired
`Export` CLI arm, `pub mod export` in lib.rs.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator → `--run-id` | typed i64 parsed by clap (V5); bound as a SQL parameter, never interpolated |
| export → SQLite | `materialize_suspected` runs on `export_write_conn` touching only `suspected_spammer` (single-writer invariant preserved for score/signal/pubkey) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-09 | Tampering | `export` INSERT…SELECT / DELETE (run_id, τ, exported_at) | mitigate | `params![run_id, tau, now]` / `?N` binding only — never `format!` into SQL (T-01-01); the SELECT predicate is a constant `suspected = 1` |
| T-05-10 | Tampering | `export_write_conn` second write connection | mitigate | touches ONLY `suspected_spammer` (never score/signal/pubkey) — single-writer invariant for the actor's tables preserved (Plan-01 contract) |
| T-05-11 | Repudiation | a `suspected_spammer` row traceable to its τ + weights | mitigate | τ stamped inline from the run's `config_json` snapshot; the weight snapshot is the run_id→config_json join (D-06); the row is point-in-time per run |
| T-05-12 | Information Disclosure | duplicating evidence into the export table | mitigate | evidence is NOT copied into `suspected_spammer` — it stays in `signal`, JOINed at read time (D-05); only verdict columns (pubkey, score, τ, rank) are materialized |
</threat_model>
