  <behavior>
    bfs.Level (LEVEL-01):
    - Seed gets level 0; its followees get level 1; their new followees level 2; etc. (shortest follow-hop distance).
    - A node enters the levels map exactly once, at first discovery (shallowest wins — first-reached BFS invariant). A node already leveled is never re-leveled to a deeper level.
    - Only followees absent from levels join the next frontier (visited-set), so a cycle A→B→A terminates.
    - Termination: loop stops when the next frontier is empty OR the next level would exceed maxLevel (D-03 cap). maxLevel<=0 or a sentinel means "no cap" — pick one and document it.
    - Returns levels{uid->int}, adjacency{F-uid->[]T-uid} (every materialized follows edge), pubkeys{uid->pubkey}.
    - The frontier expander is INJECTED (func value or interface) so the test feeds a fake graph with no live Dgraph.

    score.Score (SCORE-01, SCORE-02, D-02):
    - For each edge F->T in adjacency: if level(F) < level(T), increment vfc[T]. Strict `<` only.
    - Same-level (level(F)==level(T)) and deeper (level(F)>level(T)) followers are NOT counted (SCORE-02).
    - A target T present in adjacency but absent from levels (beyond the cap) is skipped, never scored (D-04).
    - Pure: no Dgraph access, no I/O.
  </behavior>
  <action>
    Create `internal/bfs/bfs.go` exposing `Level(ctx, seedUID string, expand FrontierExpander, maxLevel int) (levels map[string]int, adjacency map[string][]string, pubkeys map[string]string, err error)` where `FrontierExpander` is an injected interface/func matching the shape `func(ctx, uids []string) ([]FrontierResult, error)` (FrontierResult = uid, pubkey, follows[{uid,pubkey}]). Implement a FIFO frontier loop: start with the seed at level 0 as the level-0 frontier; for each level call expand on the current frontier, record every follows edge into adjacency and every uid→pubkey, level any not-yet-seen followee at currentLevel+1 and add it to the next frontier. Stop when the next frontier is empty or currentLevel+1 > maxLevel. Define a local `FrontierResult` type (or import a shared one) — keep the package buildable without `internal/dgraph` by defining the expander as a function type / small interface, so the dependency points dgraph→bfs at call time, not bfs→dgraph at compile time.

    Create `internal/score/score.go` exposing `Score(levels map[string]int, adjacency map[string][]string) map[string]int`, transcribed verbatim from 01-RESEARCH.md §Scoring: iterate adjacency edges, increment vfc[target] when level(follower) < level(target). Add a package-level doc comment preserving the D-02/D-04 correctness proof (a valid follower F of T has level(F) < level(T), so F was expanded before T and the F→T edge is already in the in-memory adjacency; the --max-level cap only drops level-M+1 discoveries which are never scored) so no future editor reintroduces `~follows` reverse-edge queries.
