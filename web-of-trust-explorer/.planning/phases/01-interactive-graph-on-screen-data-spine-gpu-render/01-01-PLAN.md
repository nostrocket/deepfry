<objective>
Walking Skeleton: scaffold the Vite + TypeScript app and prove the full render spine end-to-end with a SMALL synthetic graph — Dgraph-shaped data flows through the swappable `GraphTransport` interface, into structure-of-arrays typed buffers, into a single `@cosmos.gl/graph@3.0.0` GPU render that the user can pan and zoom. Also lay the Wave-0 test scaffold (vitest + the five CPU-pipeline test files) that Plans 02/03 fill against real scale.

This is the thinnest working slice that proves the architecture before the at-scale spikes (Plan 02 = GPU ceiling, Plan 03 = JSON wire + verdict) layer on. Keep the synthetic graph SMALL here (~1,000 nodes) — the 5M/30M stress generator is Plan 02's job. Do NOT build the loader, verdict instrument, auto-freeze, Run/Pause, fit, or hover here; those arrive with the scale work in Plans 02/03.

Purpose: De-risk the engine wiring and project setup so the two feasibility spikes start from a known-good spine.
Output: A runnable dev app rendering+panning a small synthetic graph, the GraphTransport interface, the SoA types, a small BA-shaped synthetic worker, the cosmos render adapter, COOP/COEP-configured Vite, and a green vitest CPU-pipeline suite.
</objective>
