<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Worker → main thread | 5M/30M synthetic typed buffers cross via Transferable (in-process; synthetic data, untrusted only in size) |
| GPU memory | Large vertex/link buffers uploaded to WebGL2 (resource-exhaustion risk) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03 | Denial of Service | 5M/30M buffer generation (18GB pool, D-08) | mitigate | Pre-sized typed arrays, Transferable zero-copy (no second copy), no per-node objects; in-degree as a single Uint32Array. Peak-heap is instrumented in Plan 03's verdict; if generation OOMs here, that IS the spike result to record. |
| T-01-04 | Tampering | Float32 link index precision | mitigate | Assert `nodeCount < MAX_NODE_INDEX` (2^24) before the Float32 link view (RESEARCH Pitfall 1); 5M < 16.7M is safe. |
| T-01-05 | Denial of Service | Motion sampler over 5M positions | mitigate | Sample a fixed ~10k-node subset only (RESEARCH Pitfall 5), never all 5M per tick. |
</threat_model>
