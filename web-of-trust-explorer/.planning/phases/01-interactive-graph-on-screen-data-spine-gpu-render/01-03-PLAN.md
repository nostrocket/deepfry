<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Dgraph :8080 → browser worker | Untrusted-sized JSON response crosses the wire into a blocking parser (parse-DoS / memory-exhaustion) |
| Worker → main thread | Remapped SoA buffers cross via Transferable |
| Read-only invariant | The tool must NEVER emit a Dgraph mutation (DeepFry data-separation rule) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06 | Denial of Service | JSON.parse of huge response (D-08, memory-doubling) | mitigate | Chunked per-page parse in the Worker, drop page string before next page, dense uint32 remap shrinks buffers, Transferable zero-copy, moderate page size (50000, halve if heap spikes). Peak heap is a first-class recorded verdict metric. |
| T-01-07 | Tampering | Read-only invariant | mitigate | Worker builds ONLY read DQL queries (`has(follows)` + `follows{uid}`); no `/mutate`, no mutation body ever constructed; gated by grep in Task 1 verify and the read-only convention from PATTERNS. |
| T-01-08 | Information Disclosure | Secrets in browser bundle | mitigate | No secrets exist — local read-only tool hits an unauthenticated localhost Dgraph; no API keys/tokens are embedded. Confirm no `.env` secret is bundled. |
| T-01-09 | Denial of Service | Float32 link index precision at scale | accept | Real dev DB << 16.7M nodes; assertion from Plan 01 still guards. |
</threat_model>
