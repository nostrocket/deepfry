<objective>
Build the standalone Go bridge: a new Go module under `web-of-trust-explorer/bridge/` (D-02)
that reads the whole DeepFry follow-graph from Dgraph over dgo gRPC (read-only), performs the
full server-side data-prep pass (hex→uint32 dense remap, in-degree, out-degree, Louvain
community IDs, activity timestamps), and serves the result as a little-endian binary frame over
HTTP chunked transfer at `GET /graph.bin`. This is the single server-side data-prep layer (D-03)
that removes the browser `JSON.parse` wall that FAILED the Phase 1 verdict.

Purpose: Replace the browser-direct JSON wire (01-03 FAIL → swap) with a server-side pipeline so
the browser receives compact binary it can fill straight into typed arrays. Implements PERF-01;
re-validates the at-scale load capability DATA-01/DATA-03 deferred here.
Output: A runnable `bridge` binary (via `make run`) whose `/graph.bin` endpoint streams the
encoded frame; curl-verifiable MAGIC header; unit-tested degree, Louvain, and frame encode.
</objective>
