# Phase 01.1 Plan 01: Go Binary-Streaming Bridge Summary

**Standalone Go bridge that reads the whole DeepFry follow-graph read-only over dgo/v210 gRPC, computes in/out-degree + array Louvain community IDs server-side, and streams a little-endian MAGIC+VERSION binary frame over HTTP chunked transfer at GET /graph.bin — removing the browser JSON.parse wall that FAILED the Phase 1 verdict.**

## Accomplishments
- New `wot-explorer-bridge` Go module with a deepfry-convention Makefile and the exact dgo/v210 + grpc v1.75.1 pins web-of-trust uses against the same live Dgraph.
- Read-only `ReadAll` that pages the whole `has(follows)` graph by uid cursor (`after:`, never offset), raises the gRPC receive cap to 256 MiB, discards each read txn inline, ports `createHexRemap` semantics to Go, and packs 32-byte binary pubkeys + i32 timestamps per dense node index.
- Server-side data-prep pass over the dense `[]uint32` edge list: O(E) in/out-degree and a hand-rolled array Louvain (no gonum interface graph model, bounded termination, documented fallback ladder).
- Little-endian section-wise binary frame encoder (4-byte-aligned u32 sections, packed pubkey table last) and a `GET /graph.bin` chunked HTTP server with CRP cross-origin + CORS, loopback-only bind, and a Flush per section.

## Decisions Made
- **Undirected Louvain for coloring:** symmetrize each directed follows edge to undirected weight and maximize undirected modularity — cheaper than directed Q and produces the cluster coloring OVER-02 needs. The in-file fallback ladder records the next steps if full-scale runtime is too slow (defer community → return zeros; degree alone still unblocks OVER-01). The Plan 03 verdict records the measured server-compute ms that decides this.
- **uid-keyed dense index + packed pubkeys:** remap on Dgraph uid for edge compactness; ship an index→pubkey table as packed 32-byte binary (halves the table vs hex, removes hex-decode work on the wire).
- **i32 timestamps:** unix seconds packed as i32 (Assumption A3); trivially widened later if needed.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Resolved the genproto ambiguous-import**
- **Found during:** Task 1 (`go mod tidy`)
- **Issue:** With only dgo/v210 + grpc as direct deps (no go-nostr/viper to pull newer versions like web-of-trust has), `go mod tidy` resolved BOTH the pre-split monolithic `google.golang.org/genproto` and the split `google.golang.org/genproto/googleapis/rpc`, producing `ambiguous import` build failures for `.../rpc/status`, `.../rpc/code`, `.../rpc/errdetails`.
- **Fix:** Added an explicit indirect-require block pinning `google.golang.org/genproto/googleapis/rpc`, `google.golang.org/protobuf v1.36.9`, and `golang.org/x/net|sys|text` to the same versions web-of-trust resolves, forcing the split module and dropping the monolith. No package install/substitution — only version pinning of already-trusted in-stack deps.
- **Files modified:** `bridge/go.mod`, `bridge/go.sum`
- **Verification:** `go mod tidy`, `go build ./...`, `go vet ./...`, `go test ./...` all green.
- **Committed in:** `40e38db` (Task 1 commit)
