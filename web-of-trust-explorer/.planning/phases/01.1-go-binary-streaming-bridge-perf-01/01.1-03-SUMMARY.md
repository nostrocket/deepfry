## Accomplishments
- **Loader (`src/ui/loader.ts`):** the `'receive'` stage now reads **Receiving bytes (MB/s)…** with a live `formatBytes(bytesSoFar) · X.X MB/s` readout, the rate derived from bytes / elapsed since the first receive update. The `'fetch'` label was generalised to `Fetching…` (no longer Dgraph-specific). The `'parse'` stage is retained for DgraphTransport — there is no "Parsing JSON" stage on the binary path. The no-percentage discipline is kept (chunked stream has no Content-Length).
- **LoadProgress + GoBridgeTransport:** `LoadProgress` gained an optional `bytesSoFar` field; `GoBridgeTransport` emits it on each `'receive'` update so the loader can show bytes + MB/s. Additive — DgraphTransport/SyntheticTransport are unaffected.
- **Verdict (`src/ui/verdict.ts`):** added `BridgeVerdictMetrics` (serverComputeMs / streamMs / decodeMs / layoutReadyMs / nodeCount / edgeCount / peakHeapBytes / peakHeapSource / optional pass + note) and `renderBridgeVerdict` which `console.table`s + panels them with a qualitative verdict (PASS / FAIL / **PENDING-HUMAN-VERIFY** until the operator records `pass`). It reuses `measurePeakHeap` unchanged and deliberately does **NOT** use the JSON path's 30M-edge / ≤30s extrapolation.
- **main.ts:** an `isBridge` branch parallel to `isDgraph` mounts the loader, times the server/stream/decode split from stage-transition timestamps (loadStart→first `receive` = server compute; first→last `receive` = stream; last `receive`→`layout` = decode), `await`s `measurePeakHeap` after render, and renders the bridge verdict. The `dgraph` and `synthetic` branches are intact; COOP/COEP (crossOriginIsolated) is retained for accurate peak-heap measurement.
- **D-06 propagation:** ROADMAP Phase 01.1 fleshed out (3/3 waves, verdict line); Phase 2 OVER-01/OVER-02 reframed as style-buffer encoding over bridge-provided degree/community arrays (client Worker superseded); Phase 3 FILT-01 reads bridge-provided timestamps and Refresh re-pulls via the bridge. REQUIREMENTS PERF-01 marked built & green with live verdict pending; DATA-01/DATA-03 re-validated at scale in code; OVER-01/OVER-02/FILT-01 + traceability + footer note the server-side input source.

## Decisions Made
- **No extrapolation on the bridge verdict.** The JSON path linearly extrapolated throughput to 30M edges and gated on ≤30s; the bridge loads the real graph directly, so the verdict is the measured server/stream/decode split + real counts + peak heap + a qualitative PASS, per the Claude's-discretion bar (usable / no swap / 60fps holds).
- **Verdict left PENDING-HUMAN-VERIFY, not fabricated.** See the checkpoint section below. No verdict numbers were invented.
- **bytesSoFar is additive and optional** so the JSON-wire and synthetic paths keep compiling and reading edges; only the binary path reads bytes/MB-s.
