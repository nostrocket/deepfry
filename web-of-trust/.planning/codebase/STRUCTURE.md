## Directory Purposes

**cmd/:**
- Purpose: Standalone CLI applications; each has a main.go entry point
- Contains: signal handling, config loading, user interaction, final reporting
- Key files: crawler/main.go (primary), clusterscan/main.go (analysis), discover-relays/main.go (setup), pubkeys/main.go (export), healthcheck/main.go (maintenance)

**pkg/crawler/:**
- Purpose: Relay management and event subscriptions
- Contains: Crawler struct with relay connection pool, kind 3 subscription logic, event validation, chunked writes
- Key files: crawler.go (main Crawler type and FetchAndUpdateFollows), chunks.go (processFollowsInChunks for >10k follow lists)

**pkg/dgraph/:**
- Purpose: Dgraph abstraction layer for all graph operations
- Contains: Client struct, upsert helpers, query builders, read-only analysis queries
- Key files: dgraph.go (core mutations: AddFollowers, RemoveFollower; core queries: GetStalePubkeys, CountPubkeys), clusterscan.go (read-only analysis: ResolvePubkeysToUIDs, ExpandTrustedSet, GetWeakBridges, ClusterBeneath)

**pkg/config/:**
- Purpose: Configuration management
- Contains: Config struct, LoadConfig, runtime persistence helpers (SaveForwardRelayURL, RemoveRelayURL)
- Key files: config.go (sole file; loads YAML from ~/deepfry/web-of-trust.yaml)

**bin/:**
- Purpose: Compiled binaries (generated by `make build`)
- Contains: Executable files (crawler, clusterscan, discover-relays, pubkeys, healthcheck)
- Generated: Yes; not committed to git

## Key File Locations

**Entry Points:**
- `cmd/crawler/main.go`: Main crawler loop; stale-feed orchestration
- `cmd/clusterscan/main.go`: Spam cluster detection; read-only analysis
- `cmd/discover-relays/main.go`: Relay discovery and testing
- `cmd/pubkeys/main.go`: Export pubkeys with followers to CSV
- `cmd/healthcheck/main.go`: Validate and clean database

**Core Logic:**
- `pkg/crawler/crawler.go`: Crawler type; relay pool; FetchAndUpdateFollows (main algorithm)
- `pkg/dgraph/dgraph.go`: Dgraph Client; schema, upserts, queries for write path
- `pkg/dgraph/clusterscan.go`: Read-only graph queries for clustering analysis
- `pkg/config/config.go`: YAML config loading; Viper integration

**Configuration:**
- `Makefile`: Build targets (build-crawler, build-clusterscan, etc.)
- `go.mod`: Dependency management (go-nostr, dgo, spf13/viper, etc.)

## Naming Conventions

**Files:**
- Snake_case for file names (e.g., `crawler.go`, `clusterscan.go`, `chunks.go`)
- Entry points always named `main.go` in their cmd/ subdirectories

**Functions & Methods:**
- PascalCase for exported functions (e.g., `NewClient`, `FetchAndUpdateFollows`, `GetStalePubkeys`)
- camelCase for unexported functions (e.g., `queryRelay`, `updateFollowsFromEvent`, `markRelayDead`)

**Types:**
- PascalCase for all types (e.g., `Crawler`, `Client`, `Config`, `relayState`, `WeakBridge`)
- Struct fields exported (PascalCase) or unexported (camelCase) as needed for JSON marshaling

**Variables & Constants:**
- camelCase for local variables and package-level vars (e.g., `eventsChan`, `pubkeysWithEvents`)
- UPPER_CASE for constants (e.g., `initialBackoff`, `maxBackoff`, `maxConsecutiveFailures`)

## Where to Add New Code

**New Feature (e.g., Event Filtering):**
- Primary code: `pkg/crawler/crawler.go` (add method to Crawler, update FetchAndUpdateFollows)
- Tests: `pkg/crawler/crawler_test.go` (create if missing)
- Config: `pkg/config/config.go` (add Config fields if needed; update LoadConfig)
- Entry point: Modify `cmd/crawler/main.go` if user-facing CLI flags are needed

**New Dgraph Query (e.g., Find Pubkeys by Follower Count):**
- Implementation: `pkg/dgraph/dgraph.go` (add method to Client, build DQL query, unmarshal response)
- For read-only analysis: Use `pkg/dgraph/clusterscan.go` if it fits the spam-detection pattern
- Tests: `pkg/dgraph/dgraph_test.go` (create if missing)

**New CLI Tool (e.g., Export Follow Graph):**
- Create: `cmd/newtool/main.go`
- Shared logic: Reuse pkg/dgraph.Client and pkg/config
- Makefile: Add build-newtool target and run-newtool target
- Entry point pattern: Load config, create Dgraph client, run main logic, report results, exit

**Utilities & Helpers:**
- Public helpers for follow-list processing: `pkg/crawler/chunks.go`
- Dgraph query builders (e.g., uidList): `pkg/dgraph/clusterscan.go` (defines shared helpers)
- Config mutations: `pkg/config/config.go`

## Special Directories

**queries/:**
- Purpose: Optional reference DQL queries (examples for debugging)
- Generated: No; manually created
- Committed: Yes
- Usage: Documentation; testing queries before integrating into code

**bin/:**
- Purpose: Hold compiled binaries
- Generated: Yes (by `make build`)
- Committed: No (in .gitignore)
- Usage: Executables deployed or run locally

## Structural Patterns

**Type Definitions:**
- `Crawler` (pkg/crawler/crawler.go): Holds relay pool, Dgraph client, timeout, debug flag, mutex for DB updates
- `Client` (pkg/dgraph/dgraph.go): Wraps dgo.Dgraph and gRPC connection; all graph ops go through Client methods
- `Config` (pkg/config/config.go): YAML-mapped struct with relay URLs, Dgraph address, cluster-scan settings
- Helper types: `relayState` (pkg/crawler), `WeakBridge` (pkg/dgraph/clusterscan.go), `ClusterNode` (pkg/dgraph/clusterscan.go)

**Initialization Patterns:**
- `NewClient(addr)` in pkg/dgraph: Creates and returns *Client
- `New(cfg)` in pkg/crawler: Creates Crawler, connects to all relays, ensures schema
- `LoadConfig()` in pkg/config: Reads YAML, applies defaults, creates config directories if needed

**Error Handling:**
- Custom error types for relay failures (subscriptionError, transportError)
- Fmt.Errorf with %w for error wrapping
- Defer-based resource cleanup (Close, Discard, Unsub)

**Testing:**
- Short tests run with `make test` (-short flag)
- Integration tests with build tag `integration` run with `make test-integration`
- Test files co-located with implementation (e.g., crawler_test.go next to crawler.go)
