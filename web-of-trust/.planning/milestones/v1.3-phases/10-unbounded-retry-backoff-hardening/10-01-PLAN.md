<objective>
Replace v1.2's four near-identical bounded 5-attempt Dgraph retry blocks in `cmd/crawler/main.go` with a single generic `retryDgraph[T]` helper that retries transient gRPC errors indefinitely, backs off 1m→2m→4m→5m (capped), interrupts mid-backoff on context cancellation, and accumulates per-call-type success-only durations for a cumulative-average log line emitted each batch. Add a `package main` unit test file proving the backoff sequence, ctx-cancel interruption, fatal-code passthrough, and transient-retry behavior deterministically via an injected sleep function.

Purpose: The crawler must survive any-length Dgraph outage without exiting (the observed failure is `count stale pubkeys failed: rpc error: code = Unavailable desc = error reading from server: EOF` killing the loop after ~2.5 min). Indefinite retry + longer backoff + clean shutdown keep the web-of-trust expansion running across outages.

Output: Refactored `cmd/crawler/main.go` and new `cmd/crawler/main_test.go`.
</objective>

| crawler → Dgraph (gRPC) | Outbound gRPC calls to Dgraph; responses (incl. error codes) cross back into the retry helper. No new untrusted input is introduced — error classification is by gRPC status code only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Denial of Service | retryDgraph indefinite loop | mitigate | Retry only on transient codes via `isDgraphTransient`; fatal codes return immediately. Backoff caps at 5m so a persistent outage produces bounded log volume (one line per ~5m), not a tight spin loop. SHUTDOWN-01 ctx-cancel select prevents an unkillable process. |
| T-10-02 | Denial of Service | mid-backoff non-cancellable wait | mitigate | The wait is a `select` over `sleepFn(delay)` and `ctx.Done()`; SIGINT/SIGTERM cancels the context and returns within the select, never blocking for the full interval (verified by TestRetryDgraph_CtxCancelMidBackoff). |
| T-10-03 | Information Disclosure | retry/metrics log lines | accept | Logs contain gRPC error strings and call-duration averages only — no secrets, pubkeys, or event payloads. Consistent with existing `log.Printf` usage; no PII. |
| T-10-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this phase — pure in-module Go refactor + test, no new dependencies. |
</threat_model>
