<objective>
Fix the 48-minute crawler hang: make `FetchAndUpdateFollows` resilient to a stuck or
half-open relay whose per-relay query goroutine never returns. Implements all four
Phase 11 requirements in a single coordinated change to `pkg/crawler`:

- HANG-02: bound `relay.Subscribe` (go-nostr `Fire()` ignores ctx) by running it in a
  child goroutine and selecting on the result vs `relayQueryContext.Done()`.
- HANG-01: stop gating dispatcher exit on `wg.Wait()` / `eventsChan` close — on
  `relayQueryContext.Done()`, non-blocking-drain buffered events and return
  `pubkeysWithEvents` with a `nil` error.
- HANG-03: on relay-query timeout with a relay's query still outstanding, close that
  relay's connection and mark it dead with a transport-class failure (in the
  single-threaded dispatcher, per CR-02).
- TEST-02: keep the existing regression test GREEN and add a partial-return test plus a
  close-on-timeout unit test; gate on `make test`.

Purpose: A single misbehaving relay must never freeze the whole crawler; every batch
must make progress within a bounded multiple of `c.timeout`.

Output: Modified `pkg/crawler/crawler.go` and `pkg/crawler/crawler_hang_test.go`, with
the full `-short` unit suite green.
</objective>
