<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| config file → Config struct | web-of-trust.yaml values (thresholds, relay lists) cross into runtime; a malformed or hostile file can set unsafe values |
| crawler → config file (write) | EjectRelayURL mutates web-of-trust.yaml via Viper |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-DOS | Denial of Service | relay_ejection_thresholds parse | mitigate | Post-unmarshal guard resets any threshold <= 0 to its hardcoded default (10/3/5) so a parse failure or zero value cannot eject every relay on first failure (Task 1) |
| T-07-TMP | Tampering | ejected_relays / relay_urls YAML write | mitigate | URLs written as slice values via `viper.Set` + `viper.WriteConfig` (no raw string concatenation into YAML); Viper handles marshalling/escaping (Task 2) |
| T-07-RACE | Tampering | concurrent EjectRelayURL writes | accept | viper.WriteConfig is not goroutine-safe, but ejection fires only from the single-threaded crawler main loop via onConnectFail; FetchAndUpdateFollows goroutines never call it. Documented; revisit only if write paths gain parallelism |
</threat_model>
