<objective>
Rewrite the relay state machine in pkg/crawler/crawler.go to deliver RELAY-01 (decay-not-reset failure counting), RELAY-02 (three-class classification + per-class threshold ejection), RELAY-03 (filterCap persistence + probe-up recovery), and LOG-01/02/03 (collapse per-relay/per-event log spam into one-line-per-state-change). Wire the ejection callback in cmd/crawler/main.go to the new config.EjectRelayURL from Plan 01.

Purpose: The current state machine resets the failure counter to zero on reconnect (flapping relays never get ejected), resets filterCap on reconnect (the 50→25→12→10 cascade re-runs every batch), ejects on the first failed reconnect (over-aggressive), and floods logs with per-relay/per-event lines. This plan fixes all four.

Output: per-class failure counters with halving decay, classification via the existing typed errors, unified threshold-driven ejection, persisted filterCaps with zero-cost probe-up recovery, collapsed structured logging, and unit tests for the pure decay/probe/classification logic.

This is surgery on a single cohesive state machine in one file; the three tasks are ordered by data flow (state + ejection → query/cap/probe → wiring + tests) and must not be reordered.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-CLS | Spoofing | relay NOTICE driving classification | accept | Classification maps existing typed errors (subscriptionError/transportError) and the established Phase 6 NOTICE substring match; a malicious relay can at most get itself ejected faster (self-harm), not poison other relays. Per-relay state is isolated |
| T-07-RACE2 | Tampering | concurrent counter/cap writes from NOTICE goroutine | mitigate | All per-class counters, successStreak, and probing use atomic.Int32/atomic.Bool; filterCap keeps the Phase 6 CAS loop; `go test -race` gates every task (Tasks 1-3) |
| T-07-PROBE | Denial of Service | probe-up rejections inflating filter_rejection toward the low threshold of 3 | mitigate | D-11 exemption: rejections while `rs.probing` is true never call markRelayDead(classFilterRej); the flag is force-cleared via `defer` on all queryRelay exits (Pitfall 3) so a cancelled probe cannot mislabel the next batch (Task 2) |
| T-07-DOS2 | Denial of Service | misconfigured zero threshold ejecting all relays | mitigate | markRelayDead and the failed-reconnect path treat threshold `<= 0` as the safe fallback (10), in addition to Plan 01's config-load guard (Task 1) |
| T-07-FWD | Denial of Service | forward relay accidentally ejected | mitigate | Forward relay failure path is self-contained and does not route through onConnectFail/markRelayDead; confirmed and documented (Task 3, Pitfall 6) |
