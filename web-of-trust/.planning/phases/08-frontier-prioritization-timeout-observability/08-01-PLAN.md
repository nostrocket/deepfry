<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| relay event → updateFollowsFromEvent → Dgraph | pubkey strings from untrusted relays; already gated by isValidHexPubkey (VALID-01) before reaching nquads |
| config file → viper unmarshal → backoff/quorum math | operator-controlled but malformed YAML can produce degenerate intervals |
| query-time arithmetic | `2^miss_count` shift and timestamp addition can overflow |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-OVF | Denial of Service | BackoffInterval `1 << missCount` | mitigate | Clamp to cap BEFORE the bit-shift when missCount would overflow; unit test covers large missCount (Task 1) |
| T-08-CFG | Denial of Service | miss_backoff / quorum config | mitigate | Post-unmarshal guard resets non-positive backoff fields to defaults (Task 1), mirroring EjectionThresholds correction |
| T-08-INJ | Tampering | next_attempt/miss_count nquads in MarkAttempted/Backfill | mitigate | Values are integers formatted with %d from trusted ts/miss_count math, never raw pubkey strings; UID-keyed mutations only (no pubkey concatenation into the new predicates). VALID-03 hex gate still rejects garbage pubkeys before stamping |
| T-08-DQL | Tampering | DQL injection in ordered/count queries | accept | Query strings interpolate only `%d` integer limits/timestamps via fmt.Sprintf — no untrusted string reaches the query body; same pattern as existing GetStalePubkeys |
| T-08-SC | Tampering | npm/pip/cargo installs | accept | No new third-party packages added this plan (Go stdlib + existing dgo/viper only); no install step |
</threat_model>
