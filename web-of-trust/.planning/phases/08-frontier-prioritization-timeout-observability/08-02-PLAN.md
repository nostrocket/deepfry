<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| relay goroutines → shared cancel()/done counter | concurrent writes to the quorum counter from many relay goroutines |
| relay EOSE/error → batch early-exit decision | a misbehaving relay flooding errors could race the quorum to trigger cancel prematurely |
| config quorum fraction → ceil() math | operator-set fraction drives the cancel threshold |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-RACE | Tampering | function-local `done atomic.Int32` quorum counter | mitigate | atomic.Int32 (lock-free, Phase 6/7 precedent); `go test -race` gate in Task 1; counter is function-local per batch, not shared across batches |
| T-08-EARLY | Denial of Service | premature cancel() from quorum=0 or tiny fraction | mitigate | Guard `c.quorum > 0 && queriedRelays > 0` before computing ceil; quorum=0 disables early exit entirely (full-timeout fallback). Config guard from Plan 01 Task 1 prevents negative/zero fraction surprises |
| T-08-CANCEL | Denial of Service | repeated cancel() calls from multiple goroutines | accept | context cancel() is idempotent and safe to call multiple times (documented Go semantics; existing post-cancel drain already handles it) |
| T-08-STALL | Denial of Service | a slow 30%+ of relays could still block if quorum never reached | mitigate | The 15s timeout (TIMEOUT-01) remains the hard backstop; quorum is an optimization on top, not a replacement for the deadline |
| T-08-SC | Tampering | npm/pip/cargo installs | accept | No new third-party packages (Go stdlib `math`, `sync/atomic` only); no install step |
</threat_model>
