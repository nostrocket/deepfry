Change the signature to `func (c *Client) BackfillNextAttempt(ctx context.Context, hitRefreshCadence int64) (int, error)` (IN-03 — this is the new genuine symbol; the caller in pkg/crawler/crawler.go is updated in plan 09-02, which depends on this plan).

Break when a page returns zero nodes; otherwise after processing, advance offset += batchSize and break when the page was a short (< batchSize) page.

    IMPORTANT pagination-with-filter caveat: because the query filters out nodes whose next_attempt was just set, mutating-then-paginating-by-offset can skip rows. To stay correct and simple, drive the loop by re-querying offset 0 each window is NOT safe either (mutated rows drop out of the filter, shifting the window). Use offset-stable iteration: collect each page's UIDs and last_attempt, immediately commit that page's mutation (separate `CommitNow: true` txn per window, mirroring AddFollowers per-window commits), then advance offset by the number of rows just MUTATED only if you query WITHOUT the next_attempt filter inside the loop is wrong too. Simplest correct approach: keep the `@filter(NOT has(next_attempt))` and DO NOT advance offset between windows — always query `first: batchSize, offset: 0`, because each committed window removes its rows from the filtered set, so the next `offset: 0` page is the next unbacked batch. Loop until a page returns zero rows. Accumulate total count across windows.
