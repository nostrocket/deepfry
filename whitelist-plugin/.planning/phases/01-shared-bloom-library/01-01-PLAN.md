<objective>
Create the `pkg/bloom` package: a `Builder` (server-side construction) and an immutable `Filter` (query + serialize), wrapping `github.com/bits-and-blooms/bloom/v3` with a portable custom big-endian binary format and a content-hash generation marker. Add the correctness test suite (round-trip, determinism, zero-false-negatives, measured FP rate, invalid-hex boundary).

Purpose: This package is the shared foundation consumed by Phase 2 (server build + `GET /bloom`) and Phase 3 (plugin load + query). It must de-risk the unverified bits-and-blooms v3.7.1 API — the library signatures were confirmed against pkg.go.dev but NOT compiled locally, so this plan compiles them and proves the round-trip and FP-rate behavior.

Output: `pkg/bloom/bloom.go`, `pkg/bloom/bloom_test.go`, plus `go.mod`/`go.sum` carrying the new dependency. All symbols listed in "Artifacts this phase produces" exist and the test suite passes.

    - `(*Filter) Contains(k [32]byte) bool` calls underlying `Test(k[:])` — alloc-free slice view (D-08). This is the 0-allocs hot path; do NOT allocate, do NOT take `[]byte(string)`, do NOT decode hex here.
    - `(*Filter) ContainsHex(s string) (bool, error)` — lenient boundary helper following the whitelist.go analog: len!=64 or decode failure reports not-present without panic; D-10 permits the lenient (false, nil) contract on the query side for hot-path safety. Decode isolated here.

    - Use `encoding/binary` with `binary.BigEndian` for every multi-byte field. Encode fpRate via `math.Float64bits` on write and `math.Float64frombits` on read (D-06).
    - `payload` is the library's `WriteTo`/`MarshalBinary` output VERBATIM. Do NOT re-store m or k — the payload self-describes them (D-06).
    - `(*Filter) WriteTo(w io.Writer) (int64, error)` writes header then payload, returning total bytes written.
    - `(*Filter) MarshalBinary() ([]byte, error)` implemented over a bytes.Buffer + WriteTo (discretion per D-09 note).
    - `ReadFilter(r io.Reader) (*Filter, error)` validates magic (mismatch -> ErrBadFormat), gates formatVersion (unknown -> ErrBadFormat or ErrUnsupportedVersion), reads fpRate and gen, reads payloadLen and bound-checks it before consuming the payload (short read -> ErrTruncated, defending against a truncated /bloom fetch per D-07), then reconstructs the underlying filter via the library's `ReadFrom`/`UnmarshalBinary` over exactly payloadLen bytes. Wrap underlying IO/decode failures with `fmt.Errorf("...: %w", err)` so callers can `errors.Is(err, ErrBadFormat)` / `errors.Is(err, ErrTruncated)`.

    Define package-level error sentinels with `var Err... = errors.New(...)` (at minimum `ErrBadFormat`, `ErrTruncated`; `ErrUnsupportedVersion` optional) and the magic/formatVersion constants.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| serialized filter bytes → ReadFilter | Untrusted/possibly-truncated bytes (in Phase 3 these arrive over HTTP from the server, possibly partially downloaded) cross into the deserializer. |
| hex string → *Hex helpers | External 64-char hex (from JSONL feeds in later phases) crosses into the key boundary. |
| go module install → build | A new third-party dependency enters the supply chain. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | ReadFilter (truncated/garbage bytes) | mitigate | Validate 4-byte magic and formatVersion, bound-check payloadLen before consuming payload; return ErrBadFormat/ErrTruncated, never panic (D-07; tested by TestReadFilterRejectsBadMagic/Truncated). |
| T-01-02 | Denial of Service | ReadFilter payloadLen | mitigate | payloadLen is bound-checked against available bytes before allocation so a malicious oversized length cannot drive an unbounded read/alloc (D-07). |
| T-01-03 | Tampering | hex boundary helpers | mitigate | Invalid/non-hex input handled at the *Hex boundary only — AddHex errors, ContainsHex reports not-present; never panics, never reaches the hot path (D-10). |
| T-01-04 | Tampering | on-disk byte order | mitigate | Hard invariant D-02: never call bitset.LittleEndian(); fixed big-endian header guarantees portable format across server (build) and plugin (load). Verified by negative grep. |
| T-01-SC | Tampering | go get bits-and-blooms/bloom/v3 | mitigate | [ASSUMED] package — blocking-human legitimacy checkpoint (Task 1) verifies authenticity on pkg.go.dev before install; not auto-approvable. |
</threat_model>
