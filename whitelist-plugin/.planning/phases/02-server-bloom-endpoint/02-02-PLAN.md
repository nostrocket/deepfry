<objective>
Wire the three producer seams from Plan 01 together in `cmd/server/main.go` (SRV-01): register an `onRefresh` callback — before `refresher.Start()` — that rebuilds a `pkg/bloom` filter from the refreshed key set sized by `cfg.BloomFPRate`, swaps it into the server via `SwapFilter`, and updates live stats via `SetStats` (D-10). Because the callback is registered before `Start()`, the initial synchronous refresh builds the first filter, so `/bloom` becomes ready in lockstep with the whitelist.

Add an end-to-end test proving the served filter reflects the current whitelist and that the ETag changes after the key set changes (ROADMAP success criteria 1 and 3).

Purpose: This is the integration step — it depends on the config field, the `SetOnRefresh` hook, and the server `SwapFilter`/`SetStats` methods all delivered in Plan 01 (Wave 1).
Output: Modified `cmd/server/main.go` and a new end-to-end test in `pkg/server/server_test.go`.
</objective>
