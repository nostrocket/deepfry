<objective>
Establish the two foundational, network-free building blocks for the bloom gate plugin: (1) additive `BloomConfig` + `LoadBloomConfig()` in `pkg/config` that reads the shared `~/deepfry/whitelist.yaml` using `bloom_`-prefixed keys (GATE-07, D-01/D-02/D-03), and (2) a new `pkg/bloomgate` package holding `BloomChecker` — a `handler.Checker` backed by an atomically-swappable `*bloom.Filter` with a cold-start "ready" gate that withholds decisions until a filter exists (GATE-01/GATE-02, D-06/D-12).

Purpose: These are the leaf dependencies of the plugin. The fetcher (Plan 02) writes into the checker's atomic pointer and closes its ready gate; `cmd/bloom/main.go` (Plan 02) loads the config and wires the checker into the reused `Handler`/`IOAdapter`. Building and testing them first lets Plan 02 focus purely on the network + persistence + wiring integration.

Output: `pkg/config/config.go` gains `BloomConfig` + `LoadBloomConfig`; new `pkg/bloomgate/checker.go` + `pkg/bloomgate/checker_test.go`.
</objective>
